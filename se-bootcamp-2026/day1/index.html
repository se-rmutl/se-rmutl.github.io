What is a Computer?
🖥️ คอมพิวเตอร์ในยุคใหม่

Computer คืออะไร?

คอมพิวเตอร์คืออุปกรณ์หรือระบบที่สามารถ รับข้อมูล → ประมวลผล → แสดงผล และ เก็บข้อมูล ได้ตามชุดคำสั่งที่กำหนด

💻
Personal Computer
Desktop, Notebook — ใช้เรียน เขียนโปรแกรม ทำงานทั่วไป
📱
Mobile Device
Smartphone, Tablet — คอมพิวเตอร์ขนาดเล็ก เชื่อมต่ออินเทอร์เน็ตตลอดเวลา
🖥️
Server
Web server, Database server — ให้บริการแก่ผู้ใช้จำนวนมากพร้อมกัน
☁️
Cloud Computer
VM, container, cloud service — ใช้ทรัพยากรผ่าน internet ตามความต้องการ
🔌
Embedded / IoT Device
Sensor, smart device, robot — อยู่ในอุปกรณ์จริงหรือเครื่องจักร
⚡
High-Performance Computer
เครื่องประมวลผลขนาดใหญ่ — ใช้กับ simulation, AI training, big data
คอมพิวเตอร์อยู่ทุกที่
ตั้งแต่สมาร์ทโฟนในกระเป๋าจนถึง Data Center ขนาดยักษ์
คอมพิวเตอร์ ≠ PC เท่านั้น
รวมถึงทุกอุปกรณ์ที่รับ → ประมวลผล → แสดงผล

แนวคิดพื้นฐาน
⚙️ IPO Model — Input → Process → Output → Storage

โมเดลการทำงานพื้นฐาน — IPO + Storage

📥

Input

ข้อมูลเข้า

ข้อมูลหรือคำสั่งที่ระบบรับเข้ามา

พิมพ์ข้อความกดปุ่มสแกน QRอัปโหลดไฟล์GPS location
⚙️

Process

ประมวลผล

การประมวลผลตาม logic หรือ business rule

ตรวจรหัสผ่านคำนวณราคาตรวจสิทธิ์
📤

Output

ผลลัพธ์

ผลลัพธ์ที่ระบบแสดงหรือส่งต่อไปยังผู้ใช้

แจ้งเตือนตาราง/รายงานใบเสร็จ
💾

Storage

เก็บข้อมูล

การเก็บข้อมูลเพื่อนำไปใช้ต่อในอนาคต

databasefile systemlogcloud storage
🔄
ทุกระบบดิจิทัล — ตั้งแต่ Line ถึงระบบ AI — ล้วนทำงานบนแนวคิด Input → Process → Output → Storage นี้

🛒 Case Study #1

การสั่งซื้อสินค้าออนไลน์

1

📥 Input — ผู้ใช้กรอกข้อมูล

เลือกสินค้า ระบุจำนวน กรอกที่อยู่จัดส่ง เลือกวิธีชำระเงิน แล้วกด "สั่งซื้อ"

2

⚙️ Process — ระบบทำงาน

ตรวจสอบ stock · คำนวณราคาและค่าส่ง · ยืนยันการชำระเงินกับ payment gateway · สร้าง order ID · แจ้ง warehouse

3

📤 Output — ผลลัพธ์ที่ผู้ใช้ได้รับ

หน้าจอแสดง "สั่งซื้อสำเร็จ" · ส่ง email/SMS ยืนยัน · แสดงหมายเลขคำสั่งซื้อ · อัปเดตสถานะจัดส่ง

4

💾 Storage — ข้อมูลที่เก็บ

บันทึก order details · payment record · delivery status · customer history · inventory update ใน database

E-Commerce Platform
Shopee · Lazada · JD Central

🔍 สิ่งที่ SE ต้องออกแบบ

• ระบบตรวจ concurrent orders
• Payment gateway integration
• Error handling เมื่อสินค้าหมด
• Transaction rollback กรณีชำระเงินล้มเหลว

📥 Input — นักศึกษาเลือกวิชา

ตรวจสิทธิ์นักศึกษา · ตรวจตารางซ้ำ · ตรวจจำนวนหน่วยกิต · ตรวจ prerequisite · ตรวจที่นั่งว่าง

3

📤 Output — แจ้งผลให้นักศึกษา

แสดงผล "ลงทะเบียนสำเร็จ" หรือ error message พร้อมสาเหตุ · อัปเดตตารางเรียน

4

💾 Storage — บันทึกข้อมูล

เก็บข้อมูลการลงทะเบียน · อัปเดตจำนวนที่นั่ง · บันทึก transaction log ไว้ตรวจสอบย้อนหลัง

Stakeholder
