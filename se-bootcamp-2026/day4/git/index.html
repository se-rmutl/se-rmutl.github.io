GitHub คืออะไร?

GitHub คือพื้นที่ออนไลน์สำหรับเก็บ Repository และแสดงผลงานของนักพัฒนาทั่วโลก

  • ใช้เก็บ Lab และ Project ตลอด 4 ปี
  • แสดงหลักฐานการเรียนรู้ (Evidence of Learning)
  • สร้างตัวตนในโลก Developer

"GitHub ไม่ใช่แค่ที่เก็บ Code แต่เป็นพื้นที่แสดงศักยภาพของวิศวกรซอฟต์แวร์"

Git vs GitHub

ประเด็น Git (Local) GitHub (Cloud)
คืออะไร โปรแกรมจัดการ Version เว็บไซต์เก็บ Repository ออนไลน์
อยู่ที่ไหน ในเครื่องคอมพิวเตอร์ของเรา บนระบบ Cloud (อินเทอร์เน็ต)
ใช้ทำอะไร บันทึกประวัติการแก้ไข แชร์ เก็บ และแสดงผลงาน
เปรียบเทียบ สมุดบันทึกในบ้าน หอสมุดระดับโลก

ทายสิ? (Interactive)

"ถ้าเราบันทึก (Commit) งานในเครื่องแล้ว แต่คอมพังทันที..."

งานที่ Commit ไว้จะไปปรากฏบน GitHub ไหม?

ไม่ปรากฏ! (NO)

เพราะการ Commit คือการบันทึกในเครื่อง (Local) ข้อมูลจะขึ้นไปบน GitHub ก็ต่อเมื่อเราใช้คำสั่ง git push เท่านั้น

Markdown: พลังแห่งเอกสาร

ทำไมต้องใช้?

เขียนง่ายเหมือน Plain Text แต่แสดงผลได้สวยงาม เป็นมาตรฐานที่ GitHub ใช้สำหรับไฟล์ README.md

  • # หัวข้อใหญ่
  • **Text** ตัวหนา
  • - List รายการลำดับ
  • [Link](url) ลิงก์เชื่อมโยง

Workshop Workflow

1. Setup Local
(VS Code)
2. Git Init
(Start Tracker)
3. Add & Commit
(Save History)
4. Create Repo
(GitHub Site)
5. Remote & Push
(Upload)

"เราจะทำ Step-by-Step ไปด้วยกัน!"

Step 0-1: แนะนำตัวกับ Git

ตรวจสอบว่ามี Git หรือยัง:

git --version

ตั้งค่าตัวตนในเครื่อง (ครั้งแรกครั้งเดียว):

git config --global user.name "ชื่อ นามสกุล"
git config --global user.email "your_email@example.com"
git config --global init.defaultBranch main

Step 2-3: สร้างโฟลเดอร์ & VS Code

cd %USERPROFILE%\Documents
mkdir se-bootcamp-portfolio
cd se-bootcamp-portfolio
code .

คำสั่ง code . คือการสั่งเปิดโฟลเดอร์ปัจจุบันด้วย VS Code ทันที

Step 4-5: สร้าง README.md

ใน VS Code
  1. คลิกขวาพื้นที่ Explorer เลือก New File
  2. ตั้งชื่อไฟล์ว่า README.md
  3. ใส่เนื้อหา "About Me" ตาม Template ที่แจก
  4. กด Ctrl + S เพื่อบันทึก
  5. กด Ctrl + Shift + V เพื่อดู Preview

Step 6: เริ่มต้น Git (Local)

สลับกลับมาที่ Command Prompt (CMD)

git init

ตรวจเช็คสถานะ:

git status

*จะเห็นไฟล์ README.md เป็นสีแดง (Untracked)

Step 7-8: บันทึกประวัติ (Add & Commit)

git add README.md
git commit -m "Create initial portfolio"

Add: นำไฟล์เข้า Staging Area (เตรียมตัว)
Commit: บันทึกประวัติลงเครื่องพร้อมข้อความ (Message)

Step 9: สร้าง Repository บน GitHub

  1. เข้าเว็บ GitHub กดปุ่ม + > New repository
  2. ตั้งชื่อ: se-bootcamp-portfolio
  3. ห้ามติ๊ก! "Add a README file" (เพราะเราสร้างในเครื่องแล้ว)
  4. กด Create repository
  5. คัดลอก URL ของ Repository (HTTPS)

Step 10-11: ส่งงานขึ้น Cloud (Push)

git remote add origin https://github.com/USERNAME/se-bootcamp-portfolio.git
git branch -M main
git push -u origin main
Tips

ถ้าระบบให้ Login ให้เลือก "Sign in via Browser" และกดยืนยันในหน้าต่างที่เด้งขึ้นมา

Step 12: แก้ไขและอัปเดต (Cycle)

เมื่อแก้ไขไฟล์ใน VS Code ครั้งถัดไป วัฏจักรคือ:

git add README.md
git commit -m "Update info"
git push

แก้ไข -> บันทึก -> ส่ง

Troubleshooting (เจอปัญหา?)

'git' is not recognized

ลองปิด CMD แล้วเปิดใหม่ หรือติดตั้ง Git for Windows อีกครั้ง

Remote already exists

ใช้คำสั่ง git remote set-url origin [URL] เพื่อแก้ไข

Push Failed

เช็คอินเทอร์เน็ต หรือตรวจว่า GitHub มีไฟล์ที่เครื่องไม่มีหรือเปล่า

คำสั่งวิเศษ

พิมพ์ git status บ่อยๆ เพื่อดูว่าตอนนี้เกิดอะไรขึ้นใน Repository

กิจกรรมที่ 1: Command Card Sort

git push
git init
git commit
git add
