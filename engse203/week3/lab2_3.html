HTML5 คืออะไร?

HTML5 คือภาษามาร์กอัปมาตรฐานเวอร์ชันล่าสุดสำหรับสร้างและนำเสนอเนื้อหาบนเวิลด์ไวด์เว็บ ถูกออกแบบมาเพื่อแก้ปัญหา, เพิ่มความสามารถ และปรับปรุงการทำงานร่วมกันระหว่างเทคโนโลยีเว็บต่างๆ

เป้าหมายหลักของ HTML5

ความสามารถใหม่ๆ

เพิ่ม Tags ใหม่ๆ สำหรับมัลติมีเดีย เช่น <video> และ <audio> รวมถึง <canvas> สำหรับการวาดกราฟิก

โครงสร้างที่สื่อความหมาย (Semantics)

แนะนำ Tags ที่ช่วยอธิบายโครงสร้างของหน้าเว็บได้ดีขึ้น เช่น <header>, <footer>, <nav>, <article>

การทำงานแบบ Offline และ Storage

รองรับการสร้างเว็บแอปพลิเคชันที่ทำงานได้แม้ไม่มีอินเทอร์เน็ต และมี API สำหรับจัดเก็บข้อมูลบนฝั่ง Client

ลดการพึ่งพา Plugin

ความสามารถด้านมัลติมีเดียและกราฟิกที่มาพร้อมกับภาษา ช่วยลดความจำเป็นในการใช้ Plugin ภายนอกอย่าง Flash

โครงสร้างเว็บด้วย HTML5 Semantic Elements

สร้างโครงสร้างเว็บที่สื่อความหมายและดีต่อ SEO

CSS3 มีอะไรใหม่?

CSS3 เป็นเวอร์ชันล่าสุดของ CSS ที่ไม่ได้มาเป็นเวอร์ชันเดียวทั้งหมด แต่แบ่งออกเป็น "โมดูล" (Modules) ต่างๆ ทำให้แต่ละส่วนสามารถพัฒนาและออกมาตรฐานได้อย่างอิสระ

ฟีเจอร์ที่สำคัญใน CSS3

  • Selectors: เพิ่ม Selectors ใหม่ๆ ที่ซับซ้อนและเฉพาะเจาะจงมากขึ้น
  • Box Model: ปรับปรุงการคำนวณขนาดด้วย `box-sizing`
  • Backgrounds & Borders: รองรับพื้นหลังหลายชั้น, การไล่ระดับสี (gradients), และมุมโค้ง (border-radius)
  • 2D/3D Transforms: สามารถย้าย, หมุน, ปรับขนาด, และบิดเบือน Element ได้
  • Transitions: สร้างการเปลี่ยนแปลงค่า CSS อย่างนุ่มนวล
  • Animations: สร้างการเคลื่อนไหวที่ซับซ้อนด้วย Keyframes
  • Flexible Box Layout (Flexbox): ระบบจัด Layout ใน 1 มิติที่ทรงพลัง
  • Grid Layout: ระบบจัด Layout ใน 2 มิติสำหรับการสร้างโครงสร้างที่ซับซ้อน

เจาะลึก CSS Selectors

Selectors คือหัวใจสำคัญของการเขียน CSS ที่ช่วยให้เราสามารถเลือก HTML elements ที่ต้องการจะจัดรูปแบบได้อย่างแม่นยำ

Pseudo-classes (:)

เลือก element ตามสถานะพิเศษของมัน

a:hover { /* เมื่อเมาส์อยู่บนลิงก์ */ }
input:focus { /* เมื่อกำลังพิมพ์ใน input */ }
li:nth-child(odd) { /* เลือก li ที่เป็นเลขคี่ */ }

Pseudo-elements (::)

ใช้สร้าง element เสมือนเพื่อจัดรูปแบบส่วนใดส่วนหนึ่งของ element ที่เลือก

p::first-line { /* จัดรูปแบบบรรทัดแรก */ }
p::before { content: "อ่าน: "; } /* เพิ่มเนื้อหาก่อนหน้า */
Attribute Selectors

เลือก element จาก attribute และค่าของมัน

/* เลือก a ที่มี attribute 'target' */
a[target] { ... }

/* เลือก a ที่มี target="_blank" */
a[target="_blank"] { ... }

/* เลือก input ที่มี type ขึ้นต้นด้วย "text" */
input[type^="text"] { ... }

/* เลือก class ที่มีคำว่า "icon-" */
[class*="icon-"] { ... }

    Interactive: The CSS Box Model

    ทำความเข้าใจองค์ประกอบของทุก Element

    Content

    Margin: 20px

    Border: 5px

    Padding: 20px

    box-sizing: border-box;

    เป็น property ที่สำคัญมาก ช่วยให้การคำนวณขนาดง่ายขึ้น โดยจะรวม `padding` และ `border` เข้าไปใน `width` และ `height` ที่เรากำหนด

    สำรวจหน่วยวัดใน CSS

    เลือกใช้หน่วยวัดให้เหมาะสมกับงาน

    Absolute Units

    px: (Pixels) เป็นหน่วยคงที่ ไม่เปลี่ยนแปลงตามขนาดอื่น

    Relative Units

    %: เทียบกับขนาดของ Parent element

    rem: (Root em) เทียบกับ font-size ของ Root element (<html>)

    vw/vh: (Viewport Width/Height) 1vw = 1% ของความกว้างหน้าจอ

    Parent Element (Font-size: 16px)
    width: 300px
    width: 50%
    width: 20rem
    width: 40vw

    Responsive Web Design คืออะไร?

    Responsive Web Design (RWD) คือแนวทางการออกแบบเว็บไซต์ที่ทำให้หน้าเว็บสามารถแสดงผลได้อย่างเหมาะสมในทุกขนาดหน้าจอและอุปกรณ์ ไม่ว่าจะเป็นคอมพิวเตอร์เดสก์ท็อป, แท็บเล็ต, หรือโทรศัพท์มือถือ

    องค์ประกอบหลักของ RWD

    • Fluid Grids: การใช้หน่วยวัดแบบยืดหยุ่น (เช่น %, vw) แทนหน่วยคงที่ (px) เพื่อให้โครงสร้างเว็บปรับขนาดตามหน้าจอ
    • Flexible Images: การทำให้รูปภาพและสื่อต่างๆ สามารถย่อขยายได้ตามขนาดของ Container
    • Media Queries: การใช้ CSS เพื่อกำหนดสไตล์ที่แตกต่างกันสำหรับขนาดหน้าจอที่แตกต่างกัน

    แนวคิด Mobile-First

    คือการเริ่มต้นออกแบบและพัฒนาสำหรับหน้าจอขนาดเล็ก (มือถือ) ก่อน แล้วค่อยๆ เพิ่มความซับซ้อนและปรับปรุง Layout สำหรับหน้าจอที่ใหญ่ขึ้น วิธีนี้ช่วยให้เราโฟกัสกับเนื้อหาที่สำคัญที่สุดและรับประกันประสบการณ์ที่ดีบนอุปกรณ์ที่มีข้อจำกัดมากที่สุด

    Transitions vs. Animations

    เลือกใช้การเคลื่อนไหวให้ถูกประเภท

    Transitions

    ใช้สำหรับการเปลี่ยนสถานะจาก A ไป B อย่างนุ่มนวล มักถูกกระตุ้นโดย User Action เช่น :hover

    ส่วนประกอบ: transition-property, transition-duration, transition-timing-function, transition-delay

    Animations

    ใช้สำหรับการสร้างการเคลื่อนไหวที่ซับซ้อน มีหลายขั้นตอน (Keyframes) และสามารถทำงานได้เองโดยไม่ต้องมี User Action

    ส่วนประกอบ: @keyframes, animation-name, animation-duration, animation-iteration-count

    Web Accessibility (A11y)

    Web Accessibility (หรือ A11y) คือแนวปฏิบัติในการออกแบบและสร้างเว็บไซต์เพื่อให้ทุกคนสามารถเข้าถึงและใช้งานได้ รวมถึงผู้ที่มีความบกพร่องทางร่างกายหรือข้อจำกัดอื่นๆ

    ทำไม A11y จึงสำคัญ?

    • เข้าถึงผู้ใช้ทุกคน: เปิดโอกาสให้ผู้ใช้กลุ่มใหญ่ขึ้นสามารถเข้าถึงเนื้อหาของคุณได้
    • ประสบการณ์ผู้ใช้ที่ดีขึ้น: การออกแบบที่คำนึงถึง A11y มักจะนำไปสู่ UX ที่ดีขึ้นสำหรับผู้ใช้ทุกคน
    • SEO: Search engines เข้าใจโครงสร้างของเว็บที่ใช้ Semantic HTML ได้ดีขึ้น
    • ข้อกฎหมาย: ในหลายประเทศมีกฎหมายบังคับให้เว็บไซต์ต้องเข้าถึงได้

    ตัวอย่างง่ายๆ ที่ทำได้ทันที

    การใช้ alt attribute ในแท็ก <img> เพื่อให้ Screen Reader สามารถอธิบายรูปภาพให้ผู้พิการทางสายตาฟังได้

    <img src="logo.png" alt="โลโก้ของบริษัท TechFlow">

    CSS Preprocessors คืออะไร?

    CSS Preprocessor คือเครื่องมือที่ช่วยให้เราสามารถเขียน CSS ด้วย синтаксис ที่มีฟีเจอร์เหมือนภาษาโปรแกรม เช่น ตัวแปร (Variables), การซ้อนกัน (Nesting), Mixins, และ Functions จากนั้นเครื่องมือจะคอมไพล์โค้ดนั้นให้กลายเป็น CSS ปกติที่เบราว์เซอร์เข้าใจได้

    ตัวอย่าง SCSS (Sassy CSS)

    SCSS Code
    $primary-color: #3498db;

    nav {
    ul {
    margin: 0;
    padding: 0;
    list-style: none;
    }

    li { display: inline-block; }

    a {
    color: $primary-color;
    &:hover {
    text-decoration: underline;
    }
    }
    }
    Compiled CSS
    nav ul {
    margin: 0;
    padding: 0;
    list-style: none;
    }

    nav li {
    display: inline-block;
    }

    nav a {
    color: #3498db;
    }

    nav a:hover {
    text-decoration: underline;
    }
