Full-Stack07/47

Full-Stack คืออะไรในมุมของวิศวกรซอฟต์แวร์

ไม่ใช่แค่ “ทำได้ทั้งหน้าเว็บและหลังบ้าน”

Full-Stack คือการเข้าใจการไหลของข้อมูลทั้งระบบ ตั้งแต่ผู้ใช้กรอกข้อมูล จนข้อมูลถูกตรวจสอบ ประมวลผล บันทึก และส่งผลลัพธ์กลับมาแสดงบนหน้าจอ

ทำไมต้องแยกชั้นของระบบ

  • พัฒนาแยกส่วนได้
  • ทดสอบง่ายขึ้น
  • เปลี่ยนเทคโนโลยีบางส่วนได้
  • ควบคุมความปลอดภัยและสิทธิ์เข้าถึงได้
  • ทีม Front-end/Back-end ทำงานร่วมกันได้ชัดเจน

Node.js26/47

const student = {
  name: "ชื่อ-นามสกุล",
  studentId: "รหัสนักศึกษา",
  os: process.platform,
  node: process.version,

  return `Hello ${name} (${studentId}) | OS: ${os} | Node: ${node}`;
}

console.log(createGreeting(student));

โค้ดนี้สอนอะไร

  • destructuring: ดึงค่าจาก object ในพารามิเตอร์
  • process.platform: อ่าน OS ที่ Node.js รันอยู่
  • process.version: อ่าน Node.js version

Git/GitHub30/47

Git กับ GitHub ต่างกันอย่างไร

Git

  • เครื่องมือ version control ในเครื่องเรา
  • บันทึกการเปลี่ยนแปลงเป็น commit
  • ย้อนดูประวัติและเปรียบเทียบไฟล์ได้
  • ใช้คำสั่ง เช่น init, add, commit, branch

GitHub

  • บริการบนเว็บสำหรับเก็บ repository ระยะไกล
  • ใช้ส่งงาน แชร์โค้ด และตรวจหลักฐาน
  • เชื่อมกับ Git ผ่าน remote origin
  • ใช้คำสั่ง push เพื่อส่ง commit ขึ้นไป

Git/GitHub32/47

คำสั่ง Git สำหรับ commit และ push ครั้งแรก

git config --global user.name "English Name"
git config --global user.email "github-email@example.com"

git init
git add .
git commit -m "feat: initialize ENGSE203 Lab 1"
git branch -M main
git remote add origin <repository-url>
git push -u origin main

จุดที่ต้องเข้าใจ

  • git add . เลือกไฟล์ที่จะบันทึก
  • git commit สร้างจุดบันทึกพร้อมข้อความ
  • main คือ branch หลัก
  • origin คือชื่อ remote GitHub
