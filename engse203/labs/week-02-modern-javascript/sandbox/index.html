Modern JavaScript Essentials

const / let และ Template Literal

กำหนดตัวแปรให้สื่อความหมายชัดเจน ใช้ const เป็นค่าเริ่มต้น และใช้ let เฉพาะค่าที่ต้องเปลี่ยนจริง ๆ

ลดการใช้ var, ลดความสับสนเรื่อง scope และอ่านข้อความสถานะได้ง่ายด้วย backtick

นับคะแนนด้วย let และแสดงผลด้วย Template Literal

const studentName = "Aom";

console.log(result);

// Aom: ผ่าน (10/10)
02

Transform data without mutating it

Destructuring, Spread และ map / filter / reduce

Dashboard ไม่ควรแก้ข้อมูลต้นฉบับโดยไม่จำเป็น เราจึงเลือกข้อมูล แปลงข้อมูล และสรุปข้อมูลด้วย array methods ที่อ่านเป็นขั้นตอน

แนวคิดใช้เมื่อผลต่อคุณภาพโค้ด
Destructuringดึงค่าจาก object หรือ arrayเข้าถึง property ชัดเจน ไม่ต้องเขียนซ้ำ
Spreadคัดลอก/ต่อยอดข้อมูลสร้างข้อมูลใหม่โดยไม่แก้ object/array เดิม
map / filter / reduceแปลง คัดเลือก สรุปข้อมูลลด loop ที่ยาวและทดสอบตรรกะได้ง่าย

TRY IT 2

สรุปเฉพาะงานที่ยังไม่เสร็จ

จากตัวแปร tasks ด้านบน ให้สร้าง todoTitles ซึ่งเป็น array ของชื่อ task ที่มีสถานะ todo เท่านั้น จากนั้นแสดงข้อความจำนวนรายการ

Hint: ใช้ filter แล้วต่อด้วย map

ดูเฉลยและผลลัพธ์ตัวอย่าง 
const todoTitles = tasks
  .filter(({ status }) => status === "todo")
  .map(({ title }) => title);

console.log(todoTitles);
console.log(`To do: ${todoTitles.length}`);

// ["ES Modules"]

TRY IT 2B

เปลี่ยนสถานะงานด้วย Destructuring และ Spread

หา task ที่มีสถานะ todo จากนั้นสร้าง object ใหม่ชื่อ startedTask ให้มีสถานะ doing โดยไม่แก้ไข tasks หรือ object เดิม

Hint: ใช้ find, destructuring และ { ...task, status: "doing" }

ดูเฉลยและผลลัพธ์ตัวอย่าง 
const nextTask = tasks.find(({ status }) => status === "todo");
const { title, ...taskInfo } = nextTask;

const startedTask = {
  ...taskInfo,
  title,
  status: "doing",
};

console.log(nextTask.status);    // todo
console.log(startedTask.status); // doing

ตัวแปร startedTask เป็น object ใหม่ จึงไม่เปลี่ยนข้อมูลของ nextTask โดยตรง

03

Make the UI explain what is happening

Async/Await และ Error Handling

การโหลด JSON หรือ API อาจช้า ล้มเหลว หรือได้ HTTP status ที่ไม่สำเร็จ หน้าที่ของโปรแกรมคือสื่อสารสถานะให้ผู้ใช้เข้าใจ ไม่ปล่อยให้หน้าเว็บว่าง

Loading

เริ่มโหลดข้อมูลและยังไม่มีผลลัพธ์

Success

ได้ข้อมูลและตรวจ response.ok ผ่าน

Error

คำขอล้มเหลวหรือ response.ok เป็น false

Finally

ทำงานหลัง try/catch เสมอ เช่น ซ่อน loading

TRY IT 3B

นับงาน To do หลังโหลดข้อมูลสำเร็จ

หลังได้ tasks จาก await response.json() ให้สร้าง todoCount แล้วแสดงข้อความ Success ว่ายังเหลืองาน To do กี่รายการ

Hint: ใช้ tasks.filter(({ status }) => status === "todo").length

const tasks = await response.json();
