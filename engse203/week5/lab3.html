Step 3: ThemeContext.jsx

Context สำหรับจัดการ Light/Dark theme และบันทึกลง LocalStorage

// src/contexts/ThemeContext.jsx
import { createContext, useContext, useState, useEffect } from 'react';

const ThemeContext = createContext();

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState(() => {
    const saved = localStorage.getItem('portfolio-theme');
    return saved || 'light';
  });

  useEffect(() => {
    localStorage.setItem('portfolio-theme', theme);
    document.documentElement.setAttribute('data-theme', theme);
  }, [theme]);

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme must be used within ThemeProvider');
  }
  return context;
}

🌟 Git Workflow Development Process

เราจะใช้ Feature Branch Workflow ซึ่งเป็นกระบวนการที่เป็นที่นิยมในการพัฒนาซอฟต์แวร์ โดยจะสร้าง Branch ใหม่สำหรับแต่ละ Feature ที่จะทำ

1. Initial Setup

สร้างโปรเจกต์, ติดตั้ง dependencies, และสร้าง Git repository

git init, git add, git commit

2. Feature Branch

สร้าง branch ใหม่สำหรับพัฒนา Projects section

git checkout -b feature/projects-section

3. Develop & Commit

พัฒนา feature และ commit การเปลี่ยนแปลงเป็นระยะๆ

git commit -m "feat: implement projects"

4. Merge to Main

เมื่อ feature เสร็จสิ้น, merge กลับเข้าสู่ main branch (ผ่าน Pull Request)

git checkout main; git merge ...

5. Repeat for other features

ทำซ้ำกระบวนการสำหรับ Contact form และ Enhancements

🚀 Build และ Deployment Guide

หลังจากพัฒนาเสร็จสิ้น ขั้นตอนสุดท้ายคือการ Build โปรเจกต์และนำขึ้น Host เพื่อให้สามารถเข้าชมออนไลน์ได้ แนะนำให้ใช้ Netlify หรือ Vercel เพราะง่ายและมีการเชื่อมต่อกับ Git ที่ดีเยี่ยม

Deploy ด้วย Netlify (Git Integration)

  1. Push โค้ดทั้งหมดขึ้น GitHub repository
  2. ไปที่ Netlify และสมัครสมาชิก
  3. คลิก "New site from Git" และเชื่อมต่อกับ GitHub
  4. ตั้งค่า Build: command: npm run build, directory: dist
  5. คลิก "Deploy site" และรอสักครู่ เว็บไซต์ของคุณก็จะออนไลน์!

Deploy ด้วย Vercel (Git Integration)

  1. Push โค้ดทั้งหมดขึ้น GitHub repository
  2. ไปที่ Vercel และ Import Project จาก Git
  3. Vercel จะตรวจจับการตั้งค่าโดยอัตโนมัติ คลิก "Deploy" ได้เลย

Deploy ด้วย GitHub Pages

  1. ติดตั้ง gh-pages และตั้งค่าใน package.json และ vite.config.js
  2. สั่ง Deploy ด้วย npm run deploy
  3. ไปที่ Settings > Pages ใน GitHub repo เพื่อเปิดใช้งาน

🆘 Troubleshooting Guide

เมื่อเจอปัญหา ลองดูวิธีแก้ไขเบื้องต้นเหล่านี้ก่อน

  • Build Errors: ลองลบ node_modules และ package-lock.json แล้วติดตั้งใหม่ด้วย npm install
  • Theme ไม่ทำงาน: ตรวจสอบว่า ThemeProvider ครอบ App ใน main.jsx หรือไม่ และเช็ค CSS Variables
  • Git มีปัญหา: หากต้องการย้อนกลับไป commit ก่อนหน้า ใช้ git reset --hard HEAD~1 (ระวัง! การเปลี่ยนแปลงล่าสุดจะหายไป)
