1 • Web UI

Web UI คือจุดที่ผู้ใช้ “รับรู้ระบบ”

ผู้ใช้ไม่เห็น function, route หรือ database โดยตรง ผู้ใช้เห็น ข้อความ ปุ่ม ช่องกรอก ลำดับเนื้อหา และ feedback ดังนั้น UI ที่ดีต้องตอบ 4 คำถามได้เร็ว: อยู่ที่ไหน, ทำอะไรได้, ต้องกรอกอะไร, ระบบตอบกลับอย่างไร

ก่อน/หลังปรับ: hierarchy, label, spacing, feedback, focus

Interactive: UI ที่ดีต้องมีอะไร?

2 • Separation of Concerns

HTML, CSS, JavaScript ควรรับผิดชอบคนละเรื่อง

HTML

บอกความหมายและโครงสร้าง เช่น form, label, main

ถามตัวเอง: ถ้าปิด CSS ยังอ่านลำดับเนื้อหาได้ไหม?

CSS

ควบคุมภาพและ layout เช่น Grid, Flexbox, spacing, focus

ถามตัวเอง: ถ้าเปลี่ยนข้อมูล style ยังใช้ซ้ำได้ไหม?

JavaScript

จัดการ event, state, validation และ render

ถามตัวเอง: handler สั้นและเรียก function ที่มีหน้าที่เดียวหรือไม่?

หลักสำคัญ: ไม่ใส่ onclick กระจัดกระจายใน HTML แต่ใช้ id/name แล้วผูกด้วย addEventListener

5 • Responsive Layout

เนื้อหาเดียวกัน ปรับตามพื้นที่

Responsive Design ไม่ใช่การทำหน้าแยกหลายชุด แต่คือการใช้โครงสร้างเดียวกันให้ปรับ layout ตามพื้นที่และการใช้งานจริง

Breakpoint ควรมาจากเนื้อหาที่เริ่มอ่านยาก ไม่ใช่จำชื่ออุปกรณ์

Responsive Preview Lab

Desktop 
.container { width: min(100% - 2rem, 70rem); margin-inline: auto; }
.stats { display: grid; grid-template-columns: repeat(auto-fit, minmax(10rem, 1fr)); }
@media (min-width: 48rem) {
  .page-grid { grid-template-columns: minmax(0, 2fr) minmax(18rem, 1fr); }
}
