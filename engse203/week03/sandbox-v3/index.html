1 • Web UI

UI ที่ดีต้องทำให้ผู้ใช้รู้ว่า “กำลังเกิดอะไร”

Web UI คือจุดที่ผู้ใช้รับรู้ระบบผ่านข้อความ ปุ่ม ช่องกรอก ลำดับเนื้อหา และ feedback หลังการกระทำ

UI

สิ่งที่เห็นและใช้งาน เช่น heading, form, button, status

UX

ประสบการณ์ตั้งแต่เริ่มกรอกจน submit สำเร็จ

Interaction

พฤติกรรมเมื่อผู้ใช้ click/input/submit

Feedback

loading, error, success, live preview, validation

เปรียบเทียบ UI ที่มีองค์ประกอบครบแต่สื่อสารไม่ดี กับ UI ที่มี hierarchy และ feedback

ทดลอง: ปุ่มที่ไม่มี feedback vs ปุ่มที่มี feedback

2 • Separation of Concerns

HTML, CSS และ JavaScript ควรรับผิดชอบคนละเรื่อง

HTML = Meaning

กำหนดความหมายและโครงสร้าง เช่น header, main, form, label

ถามตัวเอง: ถ้าปิด CSS แล้วยังอ่านลำดับเนื้อหาได้ไหม?

CSS = Presentation

ควบคุม layout, spacing, typography, responsive และ visual state

ถามตัวเอง: style ใช้ซ้ำได้ไหมเมื่อข้อมูลเปลี่ยน?

JavaScript = Behavior

จัดการ event, state, validation และ render feedback

ถามตัวเอง: handler สั้นและเรียก function ที่มีหน้าที่เดียวไหม?

หลักสำคัญ: ไม่กระจาย onclick ใน HTML แต่กำหนด id/name แล้วผูก event ด้วย addEventListener
<form id="request-form" novalidate>
  <label for="requester-name">Requester name</label>
  <input id="requester-name" name="requesterName" />
  <button type="submit">Submit request</button>
</form>

const form = document.querySelector('#request-form');
form.addEventListener('submit', handleSubmit);
3 • Semantic HTML

เลือก element ให้ตรงกับบทบาทของข้อมูล

Semantic HTML ช่วยให้ผู้พัฒนา, browser, search engine และ assistive technology เข้าใจโครงสร้างหน้าเว็บตรงกัน

<header> ชื่อระบบ / บทนำ
<nav> เมนูหลัก
<main> เนื้อหาหลัก
<section> แบบฟอร์ม
<section> รายการผลลัพธ์
<aside> preview / tips
<footer> อ้างอิง / ข้อมูลเพิ่มเติม
ตัวอย่าง landmark ของหน้าเว็บ

Label, id, name สำคัญอย่างไร

<label for="details">Details</label>
<textarea
  id="details"
  name="details"
  aria-describedby="details-hint details-error"
  minlength="10"
  required></textarea>
<small id="details-hint">อธิบายอย่างน้อย 10 ตัวอักษร</small>
<small id="details-error" class="error"></small>
4 • CSS Layout

Box Model → Flexbox/Grid → Responsive

ทุก element คือกล่องที่มี content, padding, border, margin การใช้ box-sizing: border-box ทำให้คุมขนาดง่ายขึ้น

margin
border
padding
content
Box Model และแนวทางเลือก Flexbox/Grid

Flexbox

เหมาะกับการจัดของบนแกนเดียว เช่น ปุ่มในแถว, icon + text

Grid

เหมาะกับ layout สองมิติ เช่น form + preview, card หลายใบ

.page-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}
.actions {
  display: flex;
  flex-wrap: wrap;
  gap: .75rem;
}
5 • Responsive Layout

หน้าเดียว ปรับตามพื้นที่ ไม่ใช่แยกตามชื่ออุปกรณ์

แนวทางที่ใช้ใน LAB 3 คือ mobile-first: เริ่มจาก 1 คอลัมน์ แล้วเพิ่มคอลัมน์เมื่อพื้นที่อ่านได้เพียงพอ

เนื้อหาเดียวกันปรับตาม viewport โดยรักษาลำดับงานเดิม

สูตรที่ใช้บ่อย

.container {
  width: min(100% - 2rem, 70rem);
  margin-inline: auto;
}
.hero h1 {
  font-size: clamp(2rem, 5vw, 3.5rem);
}
.stats {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(10rem, 1fr));
}
