ภาพรวมระบบ (Overall Description)

Product Perspective

MeetingRoom Pro เป็น Web Application ที่พัฒนาขึ้นใหม่ ทำงานร่วมกับ Mobile App (iOS/Android) และเชื่อมต่อกับระบบอื่นผ่าน API เช่น LDAP/Active Directory, Email, และ Calendar (Outlook, Google Calendar)

User Characteristics

  • พนักงานทั่วไป (80%): ต้องการความรวดเร็วและความง่ายในการจอง
  • ผู้จัดการ (15%): ต้องการดูรายงานและจัดการทีม
  • ผู้ดูแลระบบ (5%): ต้องการเครื่องมือควบคุมและจัดการระบบทั้งหมด

Constraints & Assumptions

ระบบต้องทำงานบน Browsers หลัก (Chrome, Firefox, Safari, Edge), รองรับ iOS 12+ และ Android 8+, ใช้ HTTPS เสมอ, และมี Uptime 99.5% โดยตั้งอยู่บนสมมติฐานว่าองค์กรมี Network ที่เสถียรและผู้ใช้มีความรู้พื้นฐานในการใช้งานเว็บ

ข้อกำหนดของระบบ (Specific Requirements)

REQ-F001: User Registration and Authentication

High

ระบบต้องอนุญาตให้ผู้ใช้สมัครสมาชิกและเข้าสู่ระบบได้อย่างปลอดภัยผ่าน Email/Password และ SSO พร้อมระบบ Verification และ Account Lockout

REQ-F002: Room Search and Availability Check

High

ผู้ใช้ต้องสามารถค้นหาห้องประชุมที่ว่างตามเงื่อนไข (วันที่, เวลา, ขนาด, อุปกรณ์) และเห็นสถานะห้องแบบ Real-time โดยผลการค้นหาต้องแสดงภายใน 3 วินาที

REQ-F003: Room Booking and Management

High

ผู้ใช้ต้องสามารถจอง, แก้ไข, และยกเลิกการจองได้ โดยระบบต้องป้องกันการจองซ้ำซ้อน (Double Booking) อย่างสมบูรณ์

REQ-F004: Notification System

Medium

ระบบต้องส่งการแจ้งเตือนยืนยัน, แก้ไข, ยกเลิก, และเตือนความจำก่อนประชุมผ่าน Email, SMS, และ Push Notification โดยผู้ใช้สามารถตั้งค่าการรับได้

REQ-F005: Reporting and Analytics

Medium

ผู้ดูแลระบบและผู้จัดการต้องสามารถสร้างรายงานการใช้งานห้องประชุมรายวัน/สัปดาห์/เดือน และดูสถิติสำคัญ เช่น อัตราการใช้งาน, ห้องยอดนิยม ได้

REQ-NF001: Performance

หน้าเว็บต้องโหลดภายใน 2-3 วินาที, การจองเสร็จสิ้นใน 5 วินาที, และระบบต้องรองรับผู้ใช้พร้อมกันอย่างน้อย 200 คน

REQ-NF002: Security

ใช้ HTTPS เสมอ, รหัสผ่านต้องปลอดภัย, มี Session Timeout, รองรับ 2FA, และข้อมูลส่วนบุคคลต้องถูกเข้ารหัส พร้อมมี Audit Trail

REQ-NF003: Usability

ผู้ใช้ใหม่ต้องเรียนรู้การจองได้ใน 5 นาที, รองรับภาษาไทย/อังกฤษ, และเป็นไปตามมาตรฐาน WCAG 2.1 AA สำหรับ Accessibility

REQ-NF004: Reliability

System uptime ต้องอย่างน้อย 99.5%, มี Automatic Failover, และสามารถกู้คืนระบบจากความล้มเหลวได้ภายใน 15 นาที
