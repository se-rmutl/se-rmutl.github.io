SRS: MeetingRoom Lite

1. Introduction

ภาพรวมและขอบเขตของโปรเจค MeetingRoom Lite สำหรับนักศึกษา

Executive Summary

MeetingRoom Lite เป็น Web Application พื้นฐานสำหรับจองห้องประชุม ออกแบบมาเพื่อให้นักศึกษาชั้นปีที่ 2 สามารถพัฒนาได้จริงภายใน 1 เดือน โดยใช้ Tech Stack ยอดนิยมอย่าง React.js, Node.js, Express และ PostgreSQL

1.1 Purpose

เอกสารนี้เป็น SRS ฉบับย่อ (Simplified SRS) ที่ออกแบบมาเพื่อให้นักศึกษาได้เรียนรู้กระบวนการพัฒนาระบบตั้งแต่ต้นจนจบ ตั้งแต่การรวบรวมความต้องการ, การวางแผน, ไปจนถึงการพัฒนาแบบ Full-stack ภายในระยะเวลาที่กำหนด

1.2 Scope

✅ In Scope (สิ่งที่ต้องทำ)

  • User System (Register, Login, Logout)
  • Room Booking (ดู, จอง, ยกเลิก)
  • Admin Panel (CRUD Rooms)
  • Simple Dashboard
  • Responsive Design

❌ Out of Scope (สิ่งที่ไม่ต้องทำ)

  • Email notifications
  • Calendar integration
  • Recurring bookings
  • Advanced reporting

5.1 ER Diagram

5.2 Table Schemas


CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    name VARCHAR(100) NOT NULL,
    role VARCHAR(20) DEFAULT 'user'
);

CREATE TABLE rooms (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    capacity INTEGER NOT NULL,
    floor INTEGER,
    is_active BOOLEAN DEFAULT true
);

CREATE TABLE bookings (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    room_id INTEGER REFERENCES rooms(id),
    booking_date DATE NOT NULL,
    start_time TIME NOT NULL,
    end_time TIME NOT NULL,
    title VARCHAR(255) NOT NULL,
    status VARCHAR(20) DEFAULT 'confirmed'
);
