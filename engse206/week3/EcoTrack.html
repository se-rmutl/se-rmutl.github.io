EcoTrack

Software Requirements Specification (SRS)

เอกสารเชิงโต้ตอบสำหรับข้อกำหนดของแอปพลิเคชันติดตาม Carbon Footprint เพื่อสนับสนุนเป้าหมาย Carbon Neutral University 2030

1. Introduction

1.1 Purpose (วัตถุประสงค์)

เอกสารนี้ระบุข้อกำหนดสำหรับแอปพลิเคชัน "EcoTrack" ซึ่งเป็นเครื่องมือสำหรับนักศึกษาและบุคลากรของมหาวิทยาลัยในการติดตาม, วิเคราะห์, และลดการปล่อยก๊าซเรือนกระจก (Carbon Footprint) ของตนเอง

1.2 Scope (ขอบเขต)

แอปพลิเคชันจะมุ่งเน้นไปที่ 3 กิจกรรมหลัก: การบันทึกการเดินทาง (Travel Tracking), การติดตามการใช้พลังงานในที่พัก (Energy Monitoring), และการดูรายงานสรุปและเปรียบเทียบข้อมูลของตนเอง

1.3 Definitions (คำนิยาม)

  • Carbon Footprint: ปริมาณการปล่อยก๊าซเรือนกระจกที่เกิดจากกิจกรรมของบุคคล
  • SRS: Software Requirements Specification
  • ผู้ใช้ (User): นักศึกษาหรือบุคลากรของมหาวิทยาลัยที่ลงทะเบียนในระบบ

2. Requirements

ข้อกำหนดต่างๆ ถูกแบ่งออกเป็น Functional (สิ่งที่ระบบต้องทำ) และ Non-Functional (ระบบต้องมีคุณสมบัติอย่างไร)

3.1 Functional Requirements

FR-001: การบันทึกการเดินทาง

High Priority

ระบบจะต้องอนุญาตให้ผู้ใช้บันทึกข้อมูลการเดินทางในแต่ละวัน โดยสามารถเลือกประเภทการเดินทาง (เช่น เดิน, จักรยาน, รถไฟฟ้า, รถยนต์ส่วนตัว) และระบุระยะทางเป็นกิโลเมตรได้

Rationale: เพื่อคำนวณ Carbon Footprint ที่เกิดจากการเดินทางซึ่งเป็นกิจกรรมหลัก

FR-002: การบันทึกการใช้ไฟฟ้า

High Priority

ระบบจะต้องมีฟังก์ชันให้ผู้ใช้กรอกข้อมูลการใช้พลังงานไฟฟ้าจากมิเตอร์ในที่พักอาศัยเป็นรายสัปดาห์ (หน่วยเป็น kWh)

Rationale: เพื่อคำนวณ Carbon Footprint ที่เกิดจากการใช้พลังงาน

FR-003: การแสดงผลแดชบอร์ดสรุป

Medium Priority

ระบบจะต้องแสดงผลแดชบอร์ดสรุป Carbon Footprint รายเดือนของผู้ใช้ โดยจำแนกตามประเภทกิจกรรม (การเดินทาง, การใช้พลังงาน) ในรูปแบบกราฟวงกลม (Pie Chart)

Rationale: เพื่อให้ผู้ใช้เห็นภาพรวมและเข้าใจพฤติกรรมของตนเอง

3.2 Non-Functional Requirements

NFR-001: Performance (ประสิทธิภาพ)

High Priority

การคำนวณและแสดงผลบนหน้าแดชบอร์ดสรุป (FR-003) จะต้องใช้เวลาไม่เกิน 3 วินาที หลังจากที่ผู้ใช้เปิดหน้าดังกล่าว

Rationale: เพื่อสร้างประสบการณ์การใช้งานที่ดีและรวดเร็ว

4. Acceptance Criteria

เกณฑ์การยอมรับสำหรับ Requirement FR-001: การบันทึกการเดินทาง

Scenario 1: บันทึกสำเร็จ

  • Given: ผู้ใช้อยู่ที่หน้า "บันทึกกิจกรรม"
  • When: ผู้ใช้เลือกประเภท "รถยนต์ส่วนตัว", กรอกระยะทาง "10" กม., และกด "บันทึก"
  • Then: ระบบบันทึกข้อมูล และแสดงข้อความ "บันทึกสำเร็จ"

Scenario 2: กรอกข้อมูลไม่ครบ

  • Given: ผู้ใช้อยู่ที่หน้า "บันทึกกิจกรรม"
  • When: ผู้ใช้เลือกประเภท แต่ไม่กรอกระยะทาง และกด "บันทึก"
  • Then: ระบบแสดงข้อความ "กรุณากรอกระยะทาง" และไม่บันทึกข้อมูล
