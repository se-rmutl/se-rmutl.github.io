WHY THIS CATALOG

ไม่ใช่แค่ 6 เว็บแอป แต่คือ 6 ชิ้นส่วนของระบบจริง

AUN-QA เป็นกรอบกำกับคุณภาพและจุดรวมหลักฐาน ขณะที่แต่ละกลุ่มจะได้พัฒนาระบบที่มีผู้ใช้จริง ขอบเขตชัดเจน และมีความเป็นอิสระเพียงพอให้ส่งมอบในภาคเรียนนี้ได้

✓

ใช้ได้จริง

ทุกหัวข้อมีผู้ใช้หลักและ workflow ที่ช่วยลดงานหรือแก้ปัญหาในการบริหารหลักสูตร

⌘

แยกส่งได้

แต่ละทีมมีฐานข้อมูล หน้าจอ และฟังก์ชันหลักของตนเอง ไม่ต้องรอระบบอื่นจึงใช้งานได้

↔

เชื่อมต่อได้

เตรียมข้อมูลออกเป็น CSV / JSON และระบุจุดเชื่อมต่อ API เพื่อรองรับการบูรณาการในอนาคต

▣

Evidence Package / API

ข้อมูลจากแต่ละระบบไม่จำเป็นต้องเชื่อมแบบ real-time ในระยะแรก แต่ต้องออกแบบให้ “ส่งต่อได้” อย่างเป็นมาตรฐาน

1. Source Systemระบบย่อยสร้างข้อมูลและหลักฐานตามงานจริง
2. Export Packageส่งออก CSV / JSON หรือชุดหลักฐานที่มี metadata
3. AUN-QA Coreรับข้อมูลเพื่อจัดหมวด ตรวจสอบช่องว่าง และติดตาม CQI
4. Quality Useสรุปผล ทำรายงาน และใช้ปรับปรุงรอบถัดไป
