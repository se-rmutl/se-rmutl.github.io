# เฉลยข้อสอบ Week 11: Privacy, PDPA & Ethics

**จำนวน:** 20 ข้อ
**เกณฑ์การผ่าน:** 15/20 คะแนน (75%)

**1. ความแตกต่างหลักระหว่าง Security และ Privacy คืออะไร?**
- [ ] Security ปกป้องสิทธิ์ของบุคคล ส่วน Privacy ปกป้องระบบ
- [x] **Security ปกป้องระบบจากการบุกรุก ส่วน Privacy คือการคุ้มครองสิทธิการควบคุมข้อมูลของเจ้าของ**
- [ ] ทั้งสองมีความหมายเหมือนกันทุกประการ
- [ ] Security ใช้กับฮาร์ดแวร์ ส่วน Privacy ใช้กับซอฟต์แวร์
> **คำอธิบาย:** Security โฟกัสไปที่การป้องกันระบบ (Admin view) แต่ Privacy โฟกัสที่การเคารพสิทธิ์และเจตจำนงของตัวบุคคลผู้เป็นเจ้าของข้อมูล (User view)

**2. สถานการณ์ใดคือตัวอย่างของระบบที่มี "Security ดี แต่ Privacy แย่"?**
- [ ] ฐานข้อมูลไม่ใส่รหัสผ่าน ทำให้แฮกเกอร์เข้ามาขโมยข้อมูลผู้ใช้ได้ง่าย
- [ ] มี Privacy Policy ที่ชัดเจน แต่ระบบล่มบ่อย
- [x] **ฐานข้อมูลเข้ารหัสอย่างแน่นหนา แต่บริษัทนำข้อมูลพฤติกรรมผู้ใช้ไปขายให้ผู้โฆษณาโดยไม่แจ้ง**
- [ ] ระบบมีช่องโหว่ SQL Injection แต่ไม่ได้เก็บข้อมูลส่วนตัวใดๆ
> **คำอธิบาย:** การเข้ารหัสข้อมูลคือ Security ที่ดี (ไม่มีใครแฮกได้) แต่การนำข้อมูลไปแสวงหาผลประโยชน์โดยผู้ใช้ไม่รับรู้คือการละเมิด Privacy

**3. ตามหลักการ CIA Triad ความเป็นส่วนตัว (Privacy) ถือเป็นส่วนขยายของมิติใด?**
- [x] **Confidentiality (ความลับ)**
- [ ] Integrity (ความถูกต้อง)
- [ ] Availability (ความพร้อมใช้)
- [ ] Non-repudiation (การห้ามปฏิเสธความรับผิด)
> **คำอธิบาย:** Privacy ขยายกรอบของความลับ (Confidentiality) โดยเพิ่มเรื่อง "อำนาจในการควบคุม" ของเจ้าของข้อมูลเข้าไปด้วย ไม่ใช่แค่ปกปิดไม่ให้คนอื่นเห็น

**4. ความเข้าใจผิดของนักพัฒนาเกี่ยวกับการเขียน "Privacy Policy ยาว 10 หน้า แล้วซ่อนเงื่อนไขไว้" ขัดกับหลักการใด?**
- [ ] Data Minimization
- [x] **Informed Consent (ความยินยอมที่ได้รับแจ้งอย่างชัดเจน)**
- [ ] Data Portability
- [ ] Right to be Forgotten
> **คำอธิบาย:** การขอ Consent ที่ถูกกฎหมายผู้ใช้ต้องรับรู้และ "เข้าใจจริงๆ" (Informed) การซ่อนเงื่อนไขในข้อความยาวๆ เป็นสิ่งที่ผิดหลักการ

**5. ใครคือ "เจ้าของข้อมูลส่วนบุคคล" (Data Subject) ที่แท้จริงตามกฎหมาย PDPA?**
- [ ] บริษัทผู้พัฒนาแอปพลิเคชัน
- [ ] ผู้ให้บริการ Cloud (เช่น AWS)
- [x] **ผู้ใช้งาน (User) ที่ข้อมูลนั้นอ้างอิงถึง**
- [ ] โปรแกรมเมอร์ผู้ดูแลฐานข้อมูล
> **คำอธิบาย:** เจ้าของข้อมูล (Data Subject) คือบุคคลธรรมดาที่ข้อมูลชุดนั้นสามารถระบุตัวตนไปถึงได้

**6. หน้าที่หลักของ Data Controller (ผู้ควบคุมข้อมูล) คืออะไร?**
- [x] **ตัดสินใจเกี่ยวกับวัตถุประสงค์และวิธีการประมวลผลข้อมูล**
- [ ] ประมวลผลข้อมูลตามคำสั่งของผู้ให้บริการเซิร์ฟเวอร์เท่านั้น
- [ ] ปกป้องข้อมูลไม่ให้แฮกเกอร์เจาะระบบเท่านั้น
- [ ] เขียนโค้ดและจัดการ Database โครงสร้างพื้นฐาน
> **คำอธิบาย:** ผู้ควบคุมข้อมูล (ซึ่งมักเป็นเจ้าของแอปพลิเคชัน) มีอำนาจตัดสินใจสูงสุดว่าจะเก็บข้อมูลอะไร เก็บเพื่ออะไร และต้องรับผิดชอบต่อ Data Subject โดยตรง

**7. หากบริษัท A ใช้บริการ Cloud ของ AWS เพื่อเก็บข้อมูลลูกค้า AWS มีสถานะเป็นอะไรตาม PDPA?**
- [ ] Data Subject
- [ ] Data Controller
- [x] **Data Processor**
- [ ] Data Protection Officer (DPO)
> **คำอธิบาย:** AWS เป็นผู้ประมวลผลข้อมูล (Data Processor) ซึ่งทำหน้าที่ดูแลและเก็บรักษาข้อมูลตาม "คำสั่ง" ของบริษัท A (Data Controller) เท่านั้น

**8. ข้อใดเป็นโทษทางอาญาที่สูงสุดตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA)?**
- [ ] ปรับ 5 ล้านบาท
- [x] **จำคุกไม่เกิน 1 ปี**
- [ ] จำคุกไม่เกิน 5 ปี
- [ ] ไม่มีโทษทางอาญา มีแต่โทษปรับ
> **คำอธิบาย:** โทษทางอาญาของ PDPA สูงสุดคือจำคุก 1 ปี (สำหรับกรณีละเมิดข้อมูลอ่อนไหวหรือเจตนาทุจริตแสวงหาผลประโยชน์) ส่วนปรับ 5 ล้านบาทคือโทษทางปกครอง

**9. หากเกิดข้อมูลรั่วไหล (Data Breach) ผู้ควบคุมข้อมูลต้องแจ้งเหตุแก่คณะกรรมการภายในเวลากี่ชั่วโมง?**
- [ ] 24 ชั่วโมง
- [ ] 48 ชั่วโมง
- [x] **72 ชั่วโมง**
- [ ] 7 วัน
> **คำอธิบาย:** PDPA กำหนดให้ต้องแจ้งเหตุข้อมูลรั่วไหลภายใน 72 ชั่วโมง นับแต่ทราบเหตุ

**10. ข้อใดต่อไปนี้จัดเป็น "ข้อมูลส่วนบุคคลอ่อนไหว" (Sensitive Data) ที่กฎหมายคุ้มครองเข้มงวดเป็นพิเศษ?**
- [ ] ชื่อ-นามสกุล
- [ ] เบอร์โทรศัพท์
- [ ] ที่อยู่ไปรษณีย์
- [x] **ข้อมูลชีวมิติ (Biometrics) เช่น ลายนิ้วมือ**
> **คำอธิบาย:** ข้อมูลชีวมิติ ศาสนา สุขภาพ พันธุกรรม จัดเป็น Sensitive Data ต้องได้รับความยินยอมโดยชัดแจ้ง (Explicit Consent) เสมอ

**11. ข้อใดคือความแตกต่างที่สำคัญระหว่าง PDPA ของไทย และ GDPR ของยุโรป?**
- [x] **GDPR ไม่มีโทษทางอาญา (จำคุก) แต่เน้นโทษปรับที่สูงมาก**
- [ ] GDPR ไม่คุ้มครองข้อมูลอ่อนไหว
- [ ] PDPA ค่าปรับแพงกว่า GDPR
- [ ] PDPA ไม่บังคับใช้กับวิศวกรซอฟต์แวร์
> **คำอธิบาย:** GDPR ของสหภาพยุโรป เน้นการปรับเงินอย่างรุนแรง (สูงสุด 4% ของรายได้บริษัททั่วโลก) แต่ไม่มีบทลงโทษจำคุกเหมือน PDPA

**12. หากคุณพัฒนาแอปพลิเคชันในไทย แต่มีผู้ใช้งานเป็นชาวยุโรป (EU) อาศัยอยู่ในยุโรป คุณต้องปฏิบัติตามกฎหมายใด?**
- [ ] PDPA เท่านั้น เพราะเซิร์ฟเวอร์อยู่ไทย
- [ ] GDPR เท่านั้น
- [x] **ต้องปฏิบัติตามทั้ง PDPA และ GDPR**
- [ ] ไม่ต้องปฏิบัติตามกฎหมายใดเลย
> **คำอธิบาย:** GDPR มีผลบังคับใช้นอกอาณาเขต (Extra-territorial Scope) หากแอปของคุณมีกลุ่มเป้าหมายหรือผู้ใช้งานที่อยู่ใน EU ต้องปฏิบัติตามด้วย

**13. แนวคิด "Privacy by Design" หมายถึงอะไร?**
- [ ] การออกแบบ UI ให้มีสีสันทึบแสง
- [x] **การออกแบบระบบให้มีความเป็นส่วนตัวตั้งแต่ขั้นตอนเริ่มต้นของสถาปัตยกรรม**
- [ ] การแปะเอกสาร Privacy Policy ไว้ในหน้าแรกของเว็บ
- [ ] การบังคับให้ผู้ใช้ใส่รหัสผ่านยาว 16 ตัวอักษร
> **คำอธิบาย:** Privacy by Design คือการฝังระบบการปกป้องสิทธิ์ผู้ใช้เข้าไปในขั้นตอนการออกแบบซอฟต์แวร์ตั้งแต่วันแรก (Requirement & Architecture)

**14. การตั้งค่าให้แอปพลิเคชันแชร์ข้อมูลผู้ใช้น้อยที่สุดตั้งแต่เริ่มใช้งาน โดยที่ผู้ใช้ไม่ต้องไปตั้งค่าปิดเอง สอดคล้องกับหลักการใดของ Privacy by Design?**
- [ ] Proactive not Reactive
- [x] **Privacy as the Default Setting**
- [ ] End-to-End Security
- [ ] Visibility and Transparency
> **คำอธิบาย:** Privacy as Default คือการทำค่าเริ่มต้นของซอฟต์แวร์ให้มีความเป็นส่วนตัวสูงสุด เพื่อปกป้องผู้ใช้ที่ไม่เก่งไอที

**15. ในมุมมองของการเขียนโค้ด (SE Implementation) การทำ Data Minimization คืออะไร?**
- [ ] การย่อขนาดไฟล์รูปภาพของผู้ใช้
- [x] **การดึงหรือเก็บเฉพาะข้อมูลที่จำเป็นต่อการใช้งานฟีเจอร์นั้นจริงๆ**
- [ ] การลบฐานข้อมูลทุกๆ 1 เดือน
- [ ] การเข้ารหัสข้อมูลทั้งหมดให้มีขนาดเล็กลง
> **คำอธิบาย:** Data Minimization หมายถึงหลักการลดการเก็บข้อมูลที่ไม่จำเป็น เช่น การหลีกเลี่ยงการใช้ `SELECT *` พร่ำเพรื่อ หรือเก็บข้อมูลโลเคชั่นถ้าฟีเจอร์ไม่ต้องการใช้

**16. การซ่อนเบอร์โทรศัพท์บนหน้าจอ UI ให้แสดงผลเป็น 081-XXX-XXXX เรียกว่าเทคนิคอะไร?**
- [ ] Data Injection
- [x] **Data Masking**
- [ ] Data Encryption
- [ ] Data Retention
> **คำอธิบาย:** Data Masking คือการพรางข้อมูลบางส่วนไม่ให้แสดงผลบนหน้าจอ ป้องกันคนแอบมอง (Shoulder Surfing)

**17. การตั้งเวลาลบข้อมูลประวัติผู้ใช้อัตโนมัติเมื่อหมดอายุหรือหมดความจำเป็น สอดคล้องกับเรื่องใดมากที่สุด?**
- [x] **Data Retention Policy**
- [ ] Data Breach Notification
- [ ] Data Portability
- [ ] Right to Rectification
> **คำอธิบาย:** นโยบายการจัดเก็บข้อมูล (Data Retention) จะกำหนดว่าข้อมูลแต่ละประเภทควรเก็บไว้นานเท่าใด เมื่อหมดความจำเป็นต้องลบทิ้งทันที

**18. ข้อมูลประเภทใดจัดเป็น "ข้อมูลส่วนบุคคลที่สามารถระบุตัวตนทางอ้อม" (Indirect Identification)?**
- [ ] เลขประจำตัวประชาชน
- [ ] รูปถ่ายหน้าตรง
- [x] **IP Address และ Cookie ID**
- [ ] อีเมลที่มีชื่อนามสกุลชัดเจน
> **คำอธิบาย:** IP Address มองเผินๆ อาจไม่รู้ว่าเป็นใคร แต่สามารถนำไปประมวลผลร่วมกับประวัติการล็อกอินเพื่อระบุตัวตนบุคคลนั้นได้ จึงถือเป็นข้อมูลส่วนบุคคลทางอ้อม

**19. ตามจรรยาบรรณวิชาชีพ ACM Code of Ethics ข้อ 1.2 "Avoid harm" ในบริบทของ Privacy ให้ความสำคัญกับเรื่องใด?**
- [x] **การละเมิดความเป็นส่วนตัวสามารถนำไปสู่ความเสียหายทั้งทรัพย์สิน จิตใจ และชื่อเสียงในชีวิตจริง**
- [ ] ระบบล่มบ่อยทำให้เกิดความเสียหายทางธุรกิจ
- [ ] การเขียนโค้ดที่ไม่มีคอมเมนต์ทำให้โปรแกรมเมอร์คนอื่นทำงานยาก
- [ ] ซอฟต์แวร์ต้องใช้ทรัพยากรเครื่องน้อยที่สุด
> **คำอธิบาย:** วิศวกรต้องตระหนักว่าข้อมูลที่รั่วไหลไป อาจทำให้เจ้าของข้อมูลถูกมิจฉาชีพหลอก ถูกแบล็กเมล หรือสูญเสียทรัพย์สินได้จริง

**20. หากเจ้านายสั่งให้แอบฝังโค้ดดึง Contact List ผู้ใช้โดยไม่บอกใน Privacy Policy คุณควรทำอย่างไรตามหลักจริยธรรม?**
- [ ] ทำตามสั่งทันทีเพราะเป็นลูกจ้าง
- [ ] ลาออกโดยไม่แจ้งให้ทราบล่วงหน้า
- [ ] แอบเขียนโค้ดให้ทำงานผิดพลาดเพื่อปกป้องผู้ใช้
- [x] **อธิบายความเสี่ยงด้านกฎหมาย (PDPA) และเสนอทางเลือกอื่นที่โปร่งใส**
> **คำอธิบาย:** การแก้ปัญหาทางจริยธรรมที่ดีที่สุดสำหรับ Professional Software Engineer คือการชี้แจงความเสี่ยงทางกฎหมายและธุรกิจให้ผู้บริหารทราบ พร้อมเสนอแนวทางที่ถูกต้องและโปร่งใส (Informed Consent)
