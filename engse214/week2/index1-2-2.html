Malware 101

Malware (Malicious Software) = ซอฟต์แวร์ที่ถูกออกแบบมาเพื่อทำลาย, ขโมย, แอบดู หรือเรียกค่าไถ่ข้อมูล

เป้าหมายหลัก

  • ขโมยข้อมูล (Credential / การเงิน)
  • ทำลายหรือแก้ไขข้อมูล
  • ปิดระบบให้ใช้งานไม่ได้
  • ยึดเครื่องเป็น Botnet

ช่องทางแพร่กระจาย

  • อีเมลแนบไฟล์ / ลิงก์ปลอม
  • ดาวน์โหลดโปรแกรมเถื่อน / Crack
  • ช่องโหว่ในระบบปฏิบัติการ / Service
  • USB / External Drive

คำถามชวนคิด

  • ถ้า นศ. เป็น Dev – โค้ดเราจะถูกใช้เป็น Malware ได้ไหม?
  • ถ้าเป็น User – เราจะแยกของจริง/ปลอมอย่างไร?

Malware Family

Malicious Software (ซอฟต์แวร์ประสงค์ร้าย) มีหลายประเภท พลิกการ์ดดูรายละเอียด:

Virus

VIRUS

"ต้องเกาะไฟล์อื่น"

  • ต้องการ Host File (.exe, .docm)
  • ต้องรอคนกดเปิดถึงทำงาน
  • แพร่เชื้อในเครื่องเดียวกัน / สื่อพกพา

Worm

WORM

"ไปเองได้ไม่ง้อคน"

  • Standalone (ไม่ต้องเกาะใคร)
  • แพร่กระจายเองผ่าน Network
  • ใช้ช่องโหว่ระบบ (Exploit)

Trojan

TROJAN

"หน้าเนื้อใจเสือ"

  • ปลอมเป็นโปรแกรมดี (Game, Utility, Crack)
  • ไม่แพร่กระจายเอง
  • เปิดประตูหลัง (Backdoor / RAT)

Ransomware

RANSOMWARE

"โจรเรียกค่าไถ่"

  • เข้ารหัสไฟล์ (Encrypt)
  • เรียกเงินแลก Key
  • มักขู่ปล่อยข้อมูล (Double Extortion)

Virus: Infection Cycle

ไวรัส = Malware ที่ต้องอาศัย Host File และ User Action

┌────────────────────────────────────────────────┐
│         VIRUS INFECTION CYCLE                  │
├────────────────────────────────────────────────┤
│  1. ผู้ใช้ดาวน์โหลดไฟล์ติดเชื้อ               │
│  2. ผู้ใช้รันไฟล์ → Virus Code เริ่มทำงาน     │
│  3. Virus ค้นหาไฟล์อื่น ๆ ในระบบ              │
│  4. แนบตัวเองเข้าไปในไฟล์เหล่านั้น            │
│  5. รอให้ไฟล์ใหม่ถูกเปิด → แพร่กระจายต่อ      │
│  6. Payload ทำงาน (ลบไฟล์, แก้ไขข้อมูล ฯลฯ)   │
└────────────────────────────────────────────────┘

ไวรัสจำนวนมากในอดีตเกาะอยู่ใน Macro ของเอกสาร Office → Dev ฝั่งเราอาจเผลอสร้างช่องทางให้ Virus ได้โดยไม่ตั้งใจ

Trojan & Remote Access Trojan (RAT)

Trojan = ซอฟต์แวร์ “หน้าเนื้อใจเสือ” ปลอมตัวเป็นโปรแกรมดี แต่มี Backdoor ซ่อนอยู่

ตัวอย่างประเภท Trojan

  • Backdoor Trojan – เปิดช่องให้เข้ามาควบคุมเครื่อง
  • Banking Trojan – ขโมยข้อมูลการเงิน/Online Banking
  • Downloader Trojan – ดาวน์โหลด Malware อื่นเพิ่ม

RAT Architecture

ATTACKER (C2 Server)  ⇄  Victim (RAT Client)

• สั่งถ่าย Screenshot
• เปิด Webcam / Mic
• ขโมยไฟล์ / Password
• ใช้เครื่องเราเป็นจุดโจมตีเครื่องอื่น

Ransomware Attack Lifecycle

เข้าใจ Phase ของ Ransomware = วางจุดตรวจจับ/ป้องกันได้ดีขึ้น

RANSOMWARE ATTACK LIFECYCLE

1. INFECTION
   - Phishing / Malicious Link / RDP exploit
   - ติดตั้ง Ransomware บนเครื่องเหยื่อ

2. EXECUTION
   - ปิด Antivirus
   - ลบ Shadow Copies / Backup ที่มองเห็น
   - เริ่มเข้ารหัสไฟล์ทีละโฟลเดอร์

3. ENCRYPTION
   - เลือกไฟล์สำคัญ (.doc, .pdf, .db, ฯลฯ)
   - สร้าง Key ต่อเครื่อง / ต่อเหยื่อ
   - เข้ารหัสแล้วลบไฟล์ต้นฉบับ

4. NOTIFICATION
   - แสดง Ransom Note
   - บอกจำนวนเงิน / วิธีจ่าย (ส่วนใหญ่เป็น Crypto)
   - ขู่ลบ Key หรือปล่อยข้อมูล

5. PAYMENT / RECOVERY
   - จ่ายเงินแล้ว “อาจจะ” ได้ Key
   - หรือไม่จ่ายแล้วต้องกู้จาก Backup เอง
หลักการสำคัญ: Backup + Segmentation + Patch + Awareness → ช่วยลด Impact ได้มากกว่าการ “ยอมจ่ายค่าไถ่”

Spyware & Keylogger

Spyware = แอบดูทุกอย่างที่เราทำ โดยที่เราไม่รู้ตัว

ประเภทยอดฮิต

  • Keylogger – บันทึกทุกปุ่มที่พิมพ์ (รวม Password)
  • Screen Recorder – แอบถ่ายหน้าจอ
  • Browser Hijacker – เปลี่ยนหน้า Home / Search
  • Mobile Spyware – ส่อง GPS, SMS, Call Log

ทำไมถึงน่ากลัว?

  • ไม่ทำให้เครื่องช้าจนรู้สึก → แอบอยู่ยาว ๆ
  • ข้อมูลการเงิน/ชีวิตส่วนตัวหลุดที = เสียหายเกินราคาเครื่อง
  • ใช้ Social Engineering + App ปลอมในมือถือได้ง่าย

Case Study: WannaCry (2017)

หนึ่งใน Ransomware Worm ที่สร้างความเสียหายไปทั่วโลก

Social Engineering (วิศวกรรมสังคม)

"Hacking the Human"

จุดอ่อนที่สุดของระบบความปลอดภัยไม่ใช่คอมพิวเตอร์ แต่คือ "มนุษย์"

Pretexting

สวมรอยเป็นคนอื่น (เช่น IT Support ธนาคาร เจ้าหน้าที่รัฐ)

Baiting

ล่อด้วยของฟรี (USB, Free Wi-Fi, QR code แจกของ)

Quid Pro Quo

แลกเปลี่ยน (ขอรหัสผ่านแลกกับ “ความช่วยเหลือ” หรือของรางวัล)

Question: ในโลกจริง นศ. เคยเจอ Social Engineering รูปแบบไหนแล้วบ้าง? (Line ปลอม, SMS ธนาคาร, โทรหลอก ฯลฯ)

Social Engineering Attack Cycle

SOCIAL ENGINEERING ATTACK CYCLE

1. RESEARCH
   - เก็บข้อมูลเหยื่อจาก Social Media / Website
   - หาเบอร์โทร, อีเมล, ตำแหน่งงาน, เพื่อนร่วมงาน

2. HOOK
   - ติดต่อครั้งแรกแบบ “ดูน่าเชื่อถือ”
   - อ้างอิงข้อมูลจริงเพื่อเพิ่มความเนียน
   - สร้างความเร่งด่วน / ความกลัว / ความโลภ

3. PLAY
   - ขอข้อมูลสำคัญ (OTP, Password, เลขบัตร)
   - ชวนกดลิงก์ / เปิดไฟล์ / ติดตั้ง App
   - โน้มน้าวให้เหยื่อทำ Action บางอย่าง

4. EXIT
   - ตัดสาย / ลบแชท / ลบร่องรอย
   - อาจทิ้ง Backdoor ไว้ใช้ในอนาคต
ป้องกัน Social Engineering = ฝึกตั้งคำถาม กับทุกข้อความที่สร้างความเร่งด่วน + ขอข้อมูลสำคัญ

Phishing vs Spear Phishing vs Whaling

ประเภท เป้าหมาย ลักษณะเด่น ตัวอย่าง
Phishing มวลชน (Mass) ข้อความทั่วไป, ไม่ระบุชื่อ, ลิงก์ปลอม SMS ปลอมธนาคาร / Email ปลอม PayPal
Spear Phishing บุคคล/กลุ่มเป้าหมายเฉพาะ ใช้ชื่อจริง, ข้อมูลงาน, เนื้อหาเฉพาะเจาะจง Email ปลอมจากหัวหน้างานถึงฝ่ายการเงิน
Whaling ผู้บริหารระดับสูง (C-Level) ดีไซน์อย่างดี, ใช้ข้อมูลธุรกิจลึก, ความเสียหายสูง โจมตี CEO/CFO เพื่อหลอกโอนเงินขนาดใหญ่
ในหลายองค์กร ความเสียหายจาก CEO Fraud / BEC สูงกว่าการถูกแฮกระบบเทคนิคเสียอีก

Case Study: CEO Fraud / BEC

Business Email Compromise (BEC) = Spear Phishing ที่เนียนระดับองค์กรเสียหายเป็นล้าน

SCENARIO:

• Attacker ปลอมโดเมน: ceo@companny.com (n ซ้ำ)
• ส่งอีเมลหา Finance Manager
  "ด่วน! ขอให้โอนเงิน 250,000 USD เพื่อดีล M&A
   อย่าบอกใคร เพราะยังไม่อนุมัติจากบอร์ด"

• Finance เชื่อว่าเป็น CEO จริง → โอนเงิน
• รู้ตัวอีกที = เงินหาย, ชื่อเสียงเสีย, ต้องสู้คดี/ประกัน

RED FLAGS:
- Domain ผิด 1 ตัว
- คำว่า “ด่วน/ห้ามบอกใคร”
- ขอให้ข้ามขั้นตอนปกติ
ถ้าองค์กรใช้ Security Policy + Workflow อนุมัติที่ชัดเจน (อย่างน้อย 2 คนอนุมัติ) → โจมตีแบบนี้จะสำเร็จยากขึ้นมาก

Advanced Persistent Threat (APT)

APT = การโจมตีระดับ “รัฐชาติ/องค์กรใหญ่” ที่เล่นยาว เนียน และเฉพาะเจาะจง

APT CHARACTERISTICS

1. TARGETED
   - เลือกเป้าหมายเฉพาะ (รัฐบาล, ธนาคาร, Critical Infra)
   - มีเป้าหมายชัดเจน (สอดแนม, ขโมย IP, ปั่นการเมือง)

2. PERSISTENT
   - อยู่ในระบบเป็นเดือน/ปี
   - วาง Backdoor หลายจุด
   - ปรับตัวตาม Defense ของเหยื่อ

3. ADVANCED
   - ใช้ 0-day exploit
   - ใช้ Malware เฉพาะงาน
   - ผสม Social Engineering + เทคนิคขั้นสูง

4. STEALTHY
   - ทำงานแบบ "low and slow"
   - เข้ารหัส C2 Traffic
   - พยายามทำให้ดูเหมือน Traffic ปกติ

Case Study: SolarWinds Supply Chain Attack

ตัวอย่าง APT ที่โจมตีผ่าน Supply Chain Software ระดับโลก

SOLARWINDS (APT29) – HIGH LEVEL

1. แฮกเกอร์เจาะเข้าไปใน Build System ของ SolarWinds
2. ฝัง Backdoor ลงใน Orion Update (ลงชื่อด้วย Cert ที่ถูกต้อง)
3. ลูกค้า ~18,000 องค์กรอัปเดตซอฟต์แวร์ “ที่คิดว่าปลอดภัย”
4. Backdoor แอบตื่นขึ้นหลังจาก Dormant ช่วงหนึ่ง
5. เชื่อมต่อ C2, เลือกเหยื่อที่มี Value สูง เพื่อเคลื่อนที่ลึกขึ้น

LESSONS:
- แม้ Vendor ที่น่าเชื่อถือก็ถูกโจมตีได้
- Supply Chain = เป้าหมายที่คุ้มค่าของ APT
- Monitoring ต้องดูพฤติกรรม ไม่ใช่แค่ “ไฟล์ติดมัลแวร์ไหม”

Defense in Depth: Layers & Controls

Layer 1–3: Physical / Perimeter / Network

  • Physical: Access Control, CCTV, ล็อกห้อง Server
  • Perimeter: Firewall, VPN, Web/Email Gateway
  • Network: Segmentation, IDS/IPS, Zero Trust Network

Layer 4–5: Host / Application

  • Host: Patch, EDR/Antivirus, Hardening
  • Application: Secure Coding, OWASP Top 10, Code Review
  • Logging & Monitoring: SIEM, Alerting Rules

Layer 6–7: Data & People

  • Data: Encryption, Backup (3-2-1 Rule), DLP
  • People/Process: Policy, Training, Incident Response Plan

Group Case Study: E-Commerce Data Breach

แตกกลุ่มวิเคราะห์เหตุการณ์ “ร้านออนไลน์ข้อมูลลูกค้ารั่วไหล”

BACKGROUND:
• บริษัท TechShop.co.th มีข้อมูลลูกค้า ~500,000 ราย
• ถูกโจมตีผ่าน Spear Phishing + ช่องโหว่ Database
• ข้อมูลส่วนตัว + ข้อมูลการเงินรั่วไหล

QUESTIONS:
1) Threat / Vulnerability / Attack Vector ที่ใช้มีอะไรบ้าง?
2) เป็น APT ไหม? เพราะอะไร?
3) Impact ต่อธุรกิจ, ลูกค้า, กฎหมาย มีอะไรบ้าง?
4) ถ้าเราเป็นทีม Security จะป้องกัน / ตรวจจับ / ตอบสนองอย่างไร?
จุดประสงค์: ฝึกมอง “ภาพรวม” ของการโจมตี ไม่ใช่ดูแต่ Technology แต่รวมเรื่อง คน + กระบวนการ ด้วย

Best Practices – สำหรับ Dev & ผู้ใช้

สำหรับ Dev / Software Engineer

  • ใช้ Parameterized Query ป้องกัน SQL Injection
  • Validate/Encode Input ทุกครั้ง ก่อนใช้บน UI / DB
  • อย่า Hard-code Secret (API Key, Password) ใน Source
  • คิด Threat Model คร่าว ๆ ให้กับ Feature สำคัญ

สำหรับ User / พฤติกรรมส่วนตัว

  • ไม่ส่ง OTP / Password ให้ใครทางแชท/โทรศัพท์
  • ระวังลิงก์จาก SMS/LINE ที่ไม่เคยขอมาก่อน
  • ใช้ Password Manager + เปิด MFA ทุกที่ที่มี
  • อัปเดต OS/แอป ให้ทันสมัยเสมอ

สรุป (Summary)

Threats

รู้จักศัตรู (Malware, Hacker, Insider, APT) และวิธีที่กระทบ CIA Triad

Human Factor

คนคือจุดอ่อน (Social Engineering, Phishing, CEO Fraud) → ต้องฝึก Habit ที่ปลอดภัย

Defense in Depth

ไม่มี Control ใด Control เดียวเอาอยู่ → ต้องใช้หลายชั้นตั้งแต่ Policy ถึง Data
