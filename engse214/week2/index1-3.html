WEEK 2

Cyber Threat Types & Malware

"รู้ทันภัยคุกคาม: จากไวรัสคอมพิวเตอร์ สู่แก๊งคอลเซ็นเตอร์"

วัตถุประสงค์การเรียนรู้

  • จำแนกประเภทภัยคุกคาม (Threats) และมัลแวร์ (Malware)
  • วิเคราะห์เทคนิค Social Engineering และ Phishing
  • เข้าใจแนวคิด APT และกรณีศึกษา SolarWinds
  • วิเคราะห์กรณีศึกษาในไทย: ATM Hacking, 9near, Call Center

ชวนคิด (Critical Thinking)

"ทำไมเทคโนโลยีทันสมัยขึ้น แต่คนถูกหลอกง่ายขึ้น?"

ปี 2023 คนไทยเสียหายจากภัยไซเบอร์กว่า 3 หมื่นล้านบาท!

Technical Security

Firewall แข็งแกร่ง, Encryption ซับซ้อน

Human Security

ความโลภ, ความกลัว, ความรู้เท่าไม่ถึงการณ์

"The weakest link is usually between the chair and the keyboard."

แผนที่ภัยคุกคาม (Threat Landscape)

ภัยคุกคามไม่ได้มีแค่ไวรัส แต่แบ่งตามเป้าหมายและวิธีการ

Malware

Virus, Worm, Trojan, Ransomware

Social Engineering

Phishing, Vishing (Call Center), Romance Scam

Network & Web

DDoS, SQL Injection, XSS

Mobile Threats

Fake Apps, Permission Abuse, SMS Scam

Physical Attack

ATM Skimming, Jackpotting

APT

State-Sponsored Attack (ระดับชาติ)

เจาะลึก Malware

Malware = Malicious + Software

ประเภท พฤติกรรมหลัก ต้องการคนช่วย?
Virus เกาะไฟล์อื่น (.exe), แพร่เมื่อเปิดไฟล์ ✅ ใช่ (ต้องรันไฟล์)
Worm แพร่กระจายเองผ่าน Network, ไม่ต้องเกาะใคร ❌ ไม่ (ไปเองได้)
Trojan ปลอมเป็นโปรแกรมดี, เปิดประตูหลัง (Backdoor) ✅ ใช่ (ต้องติดตั้ง)
Ransomware ล็อกไฟล์ เรียกค่าไถ่ ✅ ใช่ (ผ่าน Phishing/Exploit)

Case Study: WannaCry (2017)

ลักษณะการโจมตี

  • Type: Ransomware + Worm (ลูกผสม)
  • Exploit: EternalBlue (ช่องโหว่ SMBv1 ของ Windows)
  • Impact: 200,000+ เครื่อง ใน 150 ประเทศ
  • Thai Impact: โรงพยาบาล, ป้ายโฆษณาดิจิทัล

บทเรียนสำคัญ

การไม่อัปเดต Patch Windows (MS17-010) คือสาเหตุหลักที่ทำให้เสียหายวงกว้าง

Ooops, your files have been encrypted!

Send $300 worth of Bitcoin to this address:

12t9YDPgwueZ9NyMgw519p7AA8...

Time Left: 23:59:59

Mobile Malware: ภัยใกล้ตัว

โทรศัพท์มือถือคือเป้าหมายใหม่ ขโมย OTP และแอปธนาคาร

Fake Apps

แอปปลอมที่ไม่ได้อยู่ใน Store (Sideloading)

เช่น "แอปหาคู่", "แอปเงินกู้ด่วน", "แอปดูบอลฟรี"

Permission Abuse

แอปไฟฉาย ขอสิทธิ์เข้าถึง SMS, Contact, GPS

"ทำไมแอปแต่งรูป ต้องขอสิทธิ์อ่าน SMS OTP?"

Warning: Application "EasyLoan" requests:
- Read SMS (เสี่ยงโดนขโมย OTP)
- Read Contacts (เสี่ยงโดนโทรทวงหนี้เพื่อน)
- Camera/Microphone (เสี่ยงโดนแอบถ่าย)

Social Engineering (วิศวกรรมสังคม)

"จุดอ่อนที่เจาะง่ายที่สุด คือ มนุษย์"

98%

Smishing (SMS Phishing)

"คุณได้รับเงินดิจิทัล 10,000 บาท คลิกรับสิทธิ์: http://bit.ly/..."
"ไปรษณีย์ไทย: พัสดุของท่านตกค้าง กรุณาชำระภาษี..."

หลักการ: แนบ Link ให้กด -> โหลดแอปดูดเงิน

Vishing (Voice Phishing)

  • "สวัสดีครับ ผมร้อยตำรวจโท... คุณมีส่วนพัวพันการฟอกเงิน"
  • "พัสดุตกค้าง มีสิ่งผิดกฎหมาย"
  • Deepfake Voice: ปลอมเสียงลูกหลานโทรมาขอเงิน

Case Study: GSB ATM Jackpotting

การโจมตีทางกายภาพและตรรกะ (Physical & Logical)

เหตุการณ์ (2016)

ธนาคารออมสินต้องปิดตู้ ATM กว่า 3,000 ตู้ หลังถูกโจมตี

  • คนร้ายปล่อย Malware เข้าสู่ตู้ ATM (ยี่ห้อ NCR)
  • ทำให้ตู้จ่ายเงินออกมาเองเหมือน "Jackpot" แตก
  • ความเสียหาย: เงินสดหายไปกว่า 12 ล้านบาท

Lesson: อุปกรณ์ IoT/Embedded ต้องอัปเดต Firmware เสมอ

Case Study: 9near (Data Breach)

เมื่อข้อมูลคนไทย 55 ล้านคนหลุดสู่ Dark Web

ข้อมูลที่หลุด

ชื่อ-นามสกุล, ที่อยู่, วันเกิด, เบอร์โทรศัพท์, เลขบัตรประชาชน (Hash)

สาเหตุ

ช่องโหว่ในระบบของหน่วยงานรัฐ (คาดว่าเป็นระบบหมอพร้อม หรือระบบลงทะเบียนสิทธิ)

ผลกระทบ: แก๊ง Call Center มีข้อมูลเหยื่อครบถ้วน ทำให้การหลอกลวงน่าเชื่อถือขึ้น

Case Study: TrueMove H (2018)

ข้อมูลบัตรประชาชนหลุดเพราะตั้งค่า Cloud ผิดพลาด

AWS S3 Bucket Configuration
> Bucket Name: true-move-h-images > Access Control List (ACL): Public-Read (WRONG!) > Contents: - scan_id_card_001.jpg - scan_driving_license.jpg ... 46,000 files ... [!] ERROR: Sensitive data exposed to public internet without password.

สาเหตุ: Misconfiguration (ตั้งค่าผิด) ไม่ใช่การถูกแฮกด้วยเครื่องมือซับซ้อน

Supply Chain Attack: SolarWinds

เมื่อ "ยาม" กลายเป็นคนเปิดประตูให้โจร

1. Infiltration

แฮกเกอร์ (APT29) เจาะระบบบริษัท SolarWinds (ผู้ผลิตซอฟต์แวร์บริหารเครือข่าย)

2. Injection

ฝัง Backdoor ลงในไฟล์ Update ของโปรแกรม Orion

3. Distribution

ลูกค้า 18,000 ราย (รวมถึงรัฐบาลสหรัฐฯ) ดาวน์โหลด Update ไปติดตั้งเอง

"เราเชื่อใจ Software แต่ Software อาจถูกวางยา"

Cyber Hygiene: Password

รหัสผ่านที่ดีต้องใช้เวลานานในการเจาะ (Brute Force)

ความยาว/ชนิด ตัวเลขอย่างเดียว ตัวเลข + อักษรเล็ก/ใหญ่ + สัญลักษณ์
8 ตัวอักษร ทันที 5 ชั่วโมง
10 ตัวอักษร ทันที 5 วัน
12 ตัวอักษร 2 วินาที 200 ปี!
กฎเหล็ก: ยาว 12+ ตัว, ไม่ใช้คำในพจนานุกรม, ไม่ใช้ซ้ำ, เปิด 2FA เสมอ

2FA: เกราะชั้นที่ 2

ทำไมต้องเปิด 2FA (Two-Factor Authentication)?

Password

สิ่งที่เรารู้ (หลุดง่าย)

+

OTP / App

สิ่งที่เรามี (มือถือ)

=

Safe

แฮกเกอร์เข้าไม่ได้

"MFA ป้องกันการโจมตีอัตโนมัติได้ 99.9%" - Microsoft

APT (Advanced Persistent Threat)

ภัยคุกคามระดับ "จารชน" (Nation-State Actor)

// APT Characteristics: 1. Targeted: มีเป้าหมายชัดเจน (รัฐบาล, คู่แข่งทางธุรกิจ) 2. Persistent: ฝังตัวยาวนาน (Low and Slow) เพื่อขโมยข้อมูลเรื่อยๆ 3. Advanced: ใช้เทคนิคซับซ้อน หรือ Zero-day exploit 4. Stealthy: หลบหลีกการตรวจจับของ Antivirus

ตัวอย่าง: กลุ่ม Lazarus (เกาหลีเหนือ) ปล้นเงินคริปโต, กลุ่ม APT29 (รัสเซีย) จารกรรมข้อมูล

ความสะดวก vs ความปลอดภัย

Trade-off ที่เราต้องเลือก

สะดวก (Convenience)

  • Auto-login ทุกเว็บ
  • Password เดียวจำง่าย
  • ไม่มี 2FA (ขี้เกียจกด)

Result: แฮกง่ายมาก

ปลอดภัย (Security)

  • Logout ทุกครั้ง
  • Password ซับซ้อนและต่างกัน
  • ต้องกด OTP/App ยืนยันตัวตน

Result: ปลอดภัยสูง

สิ่งที่ต้องทำ (Action Plan)

สำหรับทุกคน

  • ✅ ตั้งสติก่อนคลิก (Stop-Look-Think)
  • ✅ เปิด 2FA ทุกบัญชี
  • ✅ อัปเดต OS/App เสมอ

สำหรับ Developer

  • ✅ Validate Input (กัน SQLi/XSS)
  • ✅ ไม่เก็บ Password เป็น Plain text
  • ✅ ใช้ HTTPS เสมอ

สำหรับองค์กร

  • ✅ อบรมพนักงาน (Security Awareness)
  • ✅ สำรองข้อมูล (Backup Strategy)
  • ✅ ซ้อมแผนรับมือ (Drill)

บทสรุป (Summary)

  • ภัยคุกคามมีทั้งทางเทคนิค (Malware) และจิตวิทยา (Social Engineering)
  • ในไทย Call Center และ Phishing SMS ระบาดหนักที่สุด
  • ข้อมูลส่วนตัว (PII) มีค่ามาก ห้ามให้ใครง่ายๆ
  • การป้องกันที่ดีที่สุดคือ สติ + 2FA + การอัปเดตระบบ
