2. วัตถุประสงค์การเรียนรู้ (Learning Objectives)

หลังจากทำ Case นี้ นักศึกษาควรจะสามารถ:

  • ระบุ Asset / Target ที่สำคัญในระบบโรงพยาบาลได้
  • อธิบาย Attack Vector และประเภทภัยคุกคาม (Threat / Malware Type) ที่เกี่ยวข้องกับ Ransomware ได้
  • วิเคราะห์ผลกระทบต่อ CIA Triad (Confidentiality / Integrity / Availability) จากเหตุการณ์ Ransomware ในโรงพยาบาลได้
  • ผูกเหตุการณ์เข้ากับแนวคิด Cyber Kill Chain / APT Lifecycle ในระดับภาพรวมได้
  • เสนอแนวทาง ป้องกัน / รับมือ / ฟื้นฟู เมื่อโรงพยาบาลถูกโจมตีด้วย Ransomware
3. ฉากเหตุการณ์ (Scenario)
เหตุการณ์นี้ ดัดแปลงจากเคสจริง ที่มีการโจมตีโรงพยาบาลด้วย Ransomware ในหลายประเทศ เช่น กรณี WannaCry ที่กระทบโรงพยาบาลในสหราชอาณาจักร (NHS) และหน่วยงานด้านสุขภาพอื่น ๆ ทั่วโลก

3.1 โรงพยาบาล ENG Hospital

โรงพยาบาลขนาดกลางแห่งหนึ่งในประเทศไทย ชื่อว่า ENG Hospital
เป็นโรงพยาบาลระดับจังหวัด มีบริการห้องฉุกเฉิน ห้องผ่าตัด OPD, IPD, X-ray, Lab, การเงิน, เวชระเบียน ฯลฯ

โรงพยาบาลใช้ระบบสารสนเทศหลัก ๆ เช่น:

  • HIS (Hospital Information System) – เวชระเบียน, นัดหมาย, Billing
  • LIS (Lab Information System) – ระบบแลป
  • PACS / RIS – ระบบภาพถ่ายทางรังสี
  • Database Server – เก็บข้อมูลผู้ป่วย / ประวัติการรักษา
  • File Server – เก็บไฟล์เอกสารภายในโรงพยาบาล

3.2 ภาพรวมโครงสร้างระบบ (อย่างง่าย)

- มีห้อง Data Center ขนาดเล็กภายในโรงพยาบาล
- เครื่อง PC ในแต่ละแผนกเชื่อมต่อผ่าน LAN เดียวกัน ยังไม่มีการแบ่ง Network Segment ที่ชัดเจน
- มีการเชื่อมต่อ Internet เพื่อใช้งาน Email / Web และเชื่อมต่อกับระบบภายนอกบางส่วน

3.3 เหตุการณ์ในวันโจมตี

วันศุกร์ เวลา 10:30 น.

  • เจ้าหน้าที่การเงินไม่สามารถเปิดไฟล์ Excel สรุปรายรับได้
  • ไฟล์หลายอันถูกเปลี่ยนชื่อและมีนามสกุลแปลก ๆ เช่น .locked, .enc, .wncry
  • มีหน้าต่างเด้งขึ้นมาว่าไฟล์ถูกเข้ารหัส และขอค่าไถ่เป็น Cryptocurrency

ภายใน 1–2 ชั่วโมง

  • เครื่อง PC ในแผนก ER, OPD, Lab, X-ray เริ่มมีอาการคล้ายกัน
  • ระบบ HIS ล่ม แพทย์เปิดดูประวัติคนไข้ผ่านระบบไม่ได้ ต้องกลับไปใช้กระดาษชั่วคราว
  • มีผู้ป่วยบางรายต้องเลื่อนการตรวจ / เลื่อนการผ่าตัด

ทีม IT ตรวจพบว่า

  • มีเครื่องจำนวนมากเปิดพอร์ต 445 (SMB) ไปยังเครื่องอื่นในเครือข่าย
  • Antivirus บางเครื่องเตือนพฤติกรรมคล้าย Ransomware / Worm
  • พบไฟล์ข้อความ README_FOR_DECRYPT.txt ถูกสร้างขึ้นหลายโฟลเดอร์
คำถามในใจทีมผู้บริหาร:
- จะจ่ายค่าไถ่หรือไม่?
- จะกู้ระบบกลับมาอย่างไร?
- ถ้าข้อมูลผู้ป่วยรั่วไหลออกไป จะเกิดผลกระทบอะไรบ้าง?
