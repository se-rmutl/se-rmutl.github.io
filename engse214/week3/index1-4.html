Part 1: What is Vulnerability?

Definition

Vulnerability (ช่องโหว่) คือจุดอ่อนหรือข้อบกพร่องในระบบ ซอฟต์แวร์ เครือข่าย หรือกระบวนการ ที่สามารถถูกใช้ประโยชน์ (Exploit) เพื่อทำให้เกิดความเสียหาย

สูตรความเสี่ยง (The Risk Formula):

RISK = Threat × Vulnerability × Impact

"ไม่มีช่องโหว่ = ไม่มีความเสี่ยง"

(ในทางทฤษฎี)

Vulnerability vs. Threat

Aspect Vulnerability (ช่องโหว่) Threat (ภัยคุกคาม)
นิยาม จุดอ่อนในระบบ สิ่งที่อาจก่อความเสียหาย
ตำแหน่ง ภายในระบบ (Internal) ภายนอก (External Origin)
การควบคุม แก้ไขได้ (Patch, Fix) ควบคุมไม่ได้ (ทำได้แค่ป้องกัน)
ตัวอย่าง รหัสผ่าน "admin", บั๊กในโค้ด Hacker, Malware, ไฟไหม้

Types of Vulnerabilities

Software

  • Buffer Overflow
  • SQL Injection
  • XSS / CSRF
  • Insecure Deserialization

Configuration

  • Default Credentials (admin/admin)
  • Unnecessary Services
  • Weak Encryption
  • Missing Patches

Human

  • Weak Passwords (123456)
  • Lack of Awareness
  • Social Engineering
  • Insider Threats

Part 3: Risk Assessment

กระบวนการระบุ วิเคราะห์ และประเมินความเสี่ยง


1. IDENTIFY
Assets, Threats, Vulns

2. ANALYZE
Impact x Likelihood

3. EVALUATE
Prioritize Risk

Risk Management Lifecycle

  • 1. Identify: ระบุความเสี่ยง
  • 2. Analyze: วิเคราะห์ผลกระทบ
  • 3. Evaluate: ประเมินระดับความเสี่ยง
  • 4. Treat: จัดการความเสี่ยง
  • 5. Monitor: ติดตามและทบทวน

Continuous Process

4 Ways to Treat Risk

1. Avoid (หลีกเลี่ยง)

เลิกทำกิจกรรมที่เสี่ยงไปเลย เช่น ปิด Service ที่ไม่จำเป็น, ยกเลิกโปรเจกต์ที่เสี่ยงเกินไป

2. Reduce / Mitigate (ลดความเสี่ยง)

ใช้มาตรการป้องกัน (Controls) เช่น ติดตั้ง Firewall, Patch OS, อบรมพนักงาน (วิธีที่ใช้บ่อยที่สุด)

3. Transfer (โอนถ่าย)

ให้ผู้อื่นรับความเสี่ยงแทน เช่น ซื้อประกันภัยไซเบอร์ (Cyber Insurance) หรือ Outsource

4. Accept (ยอมรับ)

เมื่อค่าใช้จ่ายในการป้องกัน สูงกว่ามูลค่าความเสียหาย หรือความเสี่ยงอยู่ในระดับที่ยอมรับได้

CVSS v3.1 Base Metrics

Exploitability Metrics

ความยากง่ายในการโจมตี
  • AVAttack Vector: Network, Adjacent, Local, Physical
  • ACAttack Complexity: Low (ง่าย), High (ยาก)
  • PRPrivileges Required: None, Low, High
  • UIUser Interaction: None, Required

Impact Metrics

ผลกระทบต่อ CIA Triad
  • CConfidentiality: ข้อมูลรั่วไหล (None/Low/High)
  • IIntegrity: ข้อมูลถูกแก้ไข (None/Low/High)
  • AAvailability: ระบบล่ม/ใช้ไม่ได้ (None/Low/High)

Case Study: Vulnerable Login Code

login.php (Vulnerable)
<?php $user = $_POST['u']; $pass = $_POST['p']; // ❌ V1: SQL Injection Risk $sql = "SELECT * FROM users WHERE u='$user' AND p='$pass'"; // ❌ V2: No Password Hashing // ❌ V3: Weak Session Management ?>

Risk Analysis Worksheet

  • Vulnerability: SQL Injection
  • Threat: External Hacker
  • Impact: High (Data Breach)
  • Risk Level: HIGH/CRITICAL
Fix: Use Prepared Statements & Hash Passwords!

Case Study: Equifax Breach (2017)

The Incident

Equifax (บริษัทข้อมูลเครดิต) ไม่ยอม Patch ช่องโหว่ Apache Struts (CVE-2017-5638) นานกว่า 2 เดือน ทั้งที่ Patch ออกมาแล้ว

The Impact

  • ข้อมูลส่วนตัวคนอเมริกัน 147 ล้านคนรั่วไหล
  • ค่าเสียหาย > $1.4 Billion (พันล้านดอลลาร์)
  • ผู้บริหารระดับสูงลาออกยกชุด

Lesson: Patch Management is Critical

Case Study: SolarWinds (2020)

Supply Chain Attack

Hacker ไม่ได้โจมตีลูกค้าโดยตรง แต่เจาะบริษัท SolarWinds แล้วฝัง Backdoor ลงใน Software Update ของผลิตภัณฑ์ Orion

Result: ลูกค้า 18,000 ราย (รวมถึงหน่วยงานรัฐบาลสหรัฐฯ) ดาวน์โหลด Update ที่มีไวรัสไปติดตั้งเองกับมือ เพราะ "เชื่อใจ" ผู้ผลิต

Lesson: Trust but Verify
