การเขียน JavaScript ครั้งแรก 📝

1. Inline JavaScript (ใน HTML Tag) - ไม่แนะนำ

<button onclick="alert('สวัสดี!')">คลิกฉัน</button>

2. Internal JavaScript (ใน <script>) - ใช้สำหรับทดลอง 

<script>
    console.log('สวัสดีจาก JavaScript!');
</script>

3. External JavaScript (ไฟล์แยก) ⭐ แนะนำ

ใน index.html (ควรวางไว้ท้ายสุดก่อนปิด </body>):

<script src="script.js"></script>

ใน script.js:

console.log('สวัสดีจากไฟล์ script.js');

let calculator = {

    add: function(num) { this.result += num; return this; },
    subtract: function(num) { this.result -= num; return this; },

ก้าวสู่ DOM Manipulation! 🌳

DOM = Document Object Model

DOM คือโครงสร้างของหน้าเว็บที่ JavaScript สามารถเข้าไปจัดการได้ เปรียบเสมือนแผนผังของบ้านที่ทำให้เรารู้ว่าอะไรอยู่ตรงไหน

ทำไม DOM สำคัญ?

  • JavaScript พื้นฐาน = เครื่องมือ
  • DOM = วิธีใช้เครื่องมือกับหน้าเว็บ
<!-- DOM เปรียบเหมือนต้นไม้ -->
<body>
    <header>
        <h1>หัวข้อ</h1>
    </header>
    <main>
        <p>เนื้อหา</p>
    </main>
</body>

DOM ทำให้ JavaScript "คุยกับ HTML" ได้!

let title = document.querySelector('.title');

// เลือกด้วย Tag
let container = document.querySelector('div');

// เลือกหลายตัวด้วย Class
let allTitles = document.querySelectorAll('.title');

Animation ด้วย JavaScript 🎬

แม้ว่า CSS Animation จะเป็นที่นิยม แต่ JavaScript ก็ให้การควบคุมที่ละเอียดกว่าผ่าน requestAnimationFrame

โค้ด (Fade In)

function fadeIn(element, duration = 1000) {
    element.style.opacity = 0;
    element.style.display = 'block';
    
    let start = performance.now();
    
    function animate(currentTime) {
        let elapsed = currentTime - start;
        let progress = Math.min(elapsed / duration, 1);
        element.style.opacity = progress;
        
        if (progress < 1) {
            requestAnimationFrame(animate);
        }
    }
    requestAnimationFrame(animate);
}

const box = document.getElementById('animated-box-37');
fadeIn(box);

Advanced Techniques: Virtual DOM (Concept)

นี่เป็นแนวคิดสำคัญที่อยู่เบื้องหลัง Framework สมัยใหม่อย่าง React.js

ปัญหาของ DOM จริง:

  • การเปลี่ยนแปลง DOM จริงโดยตรงนั้น ช้า
  • ทุกครั้งที่เปลี่ยนอะไรเล็กน้อย เบราว์เซอร์อาจต้องคำนวณ Layout และวาดหน้าจอใหม่ (Reflow/Repaint) ซึ่งใช้ทรัพยากรมาก

Virtual DOM แก้ปัญหานี้อย่างไร:

  1. สร้างสำเนา: Framework จะสร้าง "สำเนา" ของ DOM จริงเก็บไว้ในหน่วยความจำของ JavaScript (นี่คือ Virtual DOM) ซึ่งเป็นแค่ Object ธรรมดา
  2. คำนวณการเปลี่ยนแปลง: เมื่อข้อมูล (state) เปลี่ยน Framework จะสร้าง Virtual DOM ใหม่ขึ้นมาอีกชุด
  3. เปรียบเทียบ: Framework จะเปรียบเทียบ Virtual DOM ใหม่กับของเก่า เพื่อหาว่ามีอะไรเปลี่ยนแปลงไปบ้าง (ขั้นตอนนี้เร็วมากเพราะทำใน Memory)
  4. อัปเดตเฉพาะส่วน: Framework จะเข้าไปเปลี่ยนแปลง DOM จริง เฉพาะส่วนที่แตกต่าง เท่านั้น ทำให้เกิด Reflow/Repaint น้อยที่สุด

สรุป: Virtual DOM ช่วยเพิ่ม Performance โดยการลดการเปลี่ยนแปลง DOM จริงที่ไม่จำเป็น

        addClass: function(element, className) {
            element.classList.add(className);
        },
        setText: function(element, text) {
            element.textContent = text;
        }
    };
})();

    return function(...args) {
        clearTimeout(timeout);
        timeout = setTimeout(() => func.apply(this, args), delay);
    };
}
