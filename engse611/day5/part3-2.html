5/8

Backend Tasks (สิ่งที่คุณต้องทำ)

Routes

  • restaurants.js: เพิ่ม Logic การกรองข้อมูล (search, category, etc.) และสร้าง endpoint /:id
  • reviews.js: สร้าง POST /reviews endpoint เพื่อรับรีวิวใหม่
  • stats.js (server.js): สร้าง GET /stats endpoint เพื่อคำนวณและส่งข้อมูลสถิติ

Middleware & Utils

  • validation.js: เขียนกฎการตรวจสอบข้อมูลสำหรับฟอร์มรีวิวให้ครบ
  • fileManager.js: (Optional) เขียนฟังก์ชัน updateRestaurantRating (หรือทำใน `reviews.js` ก็ได้)
  • server.js: นำ routes ทั้งหมดมาเชื่อมต่อกับ app

วิธีทดสอบ: ใช้ Postman หรือ curl

ทดสอบแต่ละ Endpoint ทันทีที่เขียนเสร็จ เพื่อให้แน่ใจว่า API ทำงานถูกต้องก่อนที่จะไปทำ Frontend

# ตัวอย่างทดสอบ search
curl "http://localhost:3000/api/restaurants?search=ส้มตำ"

Frontend Tasks (สิ่งที่คุณต้องทำ)

API & Data Flow

  • api.js: เขียนฟังก์ชันสำหรับเรียก API ทั้งหมดให้ครบ
  • App.jsx: จัดการ State หลัก (filters, selectedRestaurant)
  • RestaurantList.jsx: ดึงข้อมูลร้านและจัดการสถานะ loading/error

Components

  • SearchBar & FilterPanel: ทำให้การค้นหาและกรองข้อมูลทำงานได้
  • RestaurantDetail.jsx: ดึงข้อมูลร้านตาม ID และรีวิวมาแสดง
  • ReviewForm.jsx: ทำฟอร์มสำหรับส่งรีวิวใหม่ และ Validate ข้อมูลฝั่ง Client

Tip: ทำทีละส่วน

เริ่มจากการทำให้ `RestaurantList` แสดงข้อมูลได้ก่อน จากนั้นค่อยๆ เพิ่มฟีเจอร์ค้นหา, กรอง, และหน้าดูรายละเอียด

เคล็ดลับสู่ความสำเร็จ (Tips for Success)

จำหลักการเหล่านี้ไว้เสมอเพื่อช่วยให้การทำโปรเจคราบรื่นและลดข้อผิดพลาด

DOs - สิ่งที่ควรทำ

  • เริ่มจาก Backend ก่อน: สร้าง API ให้พร้อม แล้ว Frontend จะทำงานง่ายขึ้น
  • ทดสอบบ่อยๆ: เขียนโค้ดเสร็จ 1 ส่วนเล็กๆ ให้รีบทดสอบทันที
  • Commit บ่อยๆ: ใช้ Git เพื่อบันทึกความคืบหน้าอย่างสม่ำเสมอ
  • อ่าน Error Messages: ข้อความ Error คือเพื่อนที่ดีที่สุดในการ Debug
  • ใช้ console.log: เป็นเครื่องมือที่ง่ายและทรงพลังในการตรวจสอบข้อมูล

DON'Ts - สิ่งที่ไม่ควรทำ

  • อย่าคัดลอกโค้ดโดยไม่เข้าใจ: พยายามทำความเข้าใจทีละบรรทัด
  • อย่ามุ่งหวัง Perfect ตั้งแต่แรก: ทำให้ทำงานได้ก่อน แล้วค่อยปรับปรุง (Refactor)
  • อย่าทำ Bonus Features ก่อน: ทำส่วนที่เป็น Required ให้เสร็จสมบูรณ์ก่อนเสมอ
  • อย่าลืม Push Code: Push งานขึ้น GitHub บ่อยๆ เพื่อสำรองข้อมูล
