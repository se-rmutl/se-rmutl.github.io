ทบทวนสถาปัตยกรรม (Architecture)

โปรเจคนี้เราจะใช้สถาปัตยกรรมแบบ 3-Tier (3-Tier Architecture) ที่ได้เรียนไป ซึ่งเป็นการแยกส่วนการทำงานอย่างชัดเจน

Presentation Layer

(Frontend - React)

แสดง UI, รับ Input, แสดงข้อมูล

HTTP (fetch API)

Application Layer

(Backend - Express)

จัดการ Business Logic, API, Validation

File System

Data Layer

(JSON Files)

จัดเก็บข้อมูลร้านอาหารและรีวิว

Key Principle: Frontend จะไม่ยุ่งกับไฟล์ข้อมูลโดยตรง แต่จะคุยกับ Backend ผ่าน API เท่านั้น

Backend API Endpoints

นี่คือรายการ API Endpoints ที่เราต้องสร้างในฝั่ง Express

Restaurant Endpoints

  • GET /api/restaurants: ดึงร้านทั้งหมด (พร้อมรองรับ query string สำหรับ search/filter)
  • GET /api/restaurants/:id: ดึงข้อมูลร้านเดียวตาม ID พร้อมรีวิวของร้านนั้น
  • GET /api/restaurants/random: สุ่มร้านอาหาร 1 ร้าน (Bonus)

Review & Stats Endpoints

  • POST /api/reviews: เพิ่มรีวิวใหม่สำหรับร้านอาหาร (ต้องมีการ Validate)
  • GET /api/stats: ดึงข้อมูลสถิติภาพรวม เช่น จำนวนร้าน, จำนวนรีวิวทั้งหมด, 5 ร้านที่คะแนนสูงสุด
7/20

Data Structure & Relationship

ข้อมูลของเราถูกแบ่งเป็น 2 ส่วนหลัก โดยมีความสัมพันธ์กันแบบ One-to-Many

restaurants.json

  • id: 1 (Primary Key)
  • name: "ร้านส้มตำป้าแดง"
  • category: "อาหารไทย"
  • ...
1 ∞
One-to-Many

reviews.json

  • id: 1
  • restaurantId: 1 (Foreign Key)
  • userName: "สมชาย"
  • rating: 5
  • ...

การคำนวณ Average Rating

ทุกครั้งที่มีการเพิ่มรีวิวใหม่สำหรับร้าน A, Backend จะต้องคำนวณคะแนนเฉลี่ยของรีวิวทั้งหมดที่ `restaurantId` ตรงกับ ID ของร้าน A แล้วอัปเดตข้อมูลในไฟล์ `restaurants.json`

Step 2: Backend Development (45 นาที)

สร้าง API ทั้งหมดตามโจทย์ โดยเน้นไปที่การเติมโค้ดในส่วน `TODO`

สิ่งที่ต้องทำ (เรียงตามลำดับ)

  1. `routes/restaurants.js`: เพิ่ม Logic การกรองข้อมูล (search, category, etc.) และสร้าง endpoint `/:id`
  2. `middleware/validation.js`: เขียนกฎการตรวจสอบข้อมูลสำหรับฟอร์มรีวิวให้ครบ
  3. `utils/fileManager.js`: เขียนฟังก์ชัน `updateRestaurantRating`
  4. `routes/reviews.js`: สร้าง `POST /reviews` endpoint เพื่อรับรีวิวใหม่
  5. `routes/stats.js`: สร้าง `GET /stats` endpoint เพื่อคำนวณและส่งข้อมูลสถิติ
  6. `server.js`: นำ routes ทั้งหมดมาเชื่อมต่อกับ app

วิธีทดสอบ

ใช้ **Postman** หรือ **curl** ยิง Request ไปที่แต่ละ Endpoint ทันทีที่เขียนเสร็จ เพื่อตรวจสอบการทำงานและหาข้อผิดพลาด

# ทดสอบ search
curl "http://localhost:3000/api/restaurants?search=ส้มตำ"

Step 3: Frontend Development (60 นาที)

สร้าง UI และเชื่อมต่อกับ API ที่ทำไว้ใน Step 2

แนวทางการทำ (เรียงตามลำดับ)

  1. `services/api.js`: เขียนฟังก์ชันสำหรับเรียก API ทั้งหมดให้ครบ
  2. `RestaurantList.jsx`: เริ่มจากดึงข้อมูลร้านทั้งหมดมาแสดงผลให้ได้ก่อน

  6. `ReviewForm.jsx`: ทำฟอร์มสำหรับส่งรีวิวใหม่ และ Validate ข้อมูลฝั่ง Client

Tips ✨

ใช้ `console.log()` เพื่อดูข้อมูลที่ได้รับจาก API และตรวจสอบ State ของ React Component อยู่เสมอ

Step 4: Integration & Testing (30 นาที)

เชื่อมต่อ Frontend และ Backend เข้าด้วยกัน และทดสอบการทำงานทั้งหมด

🚫 ปัญหาที่เจอบ่อยที่สุด: CORS Error

หากเจอ Error ใน Console ของเบราว์เซอร์ว่า `Access to fetch has been blocked by CORS policy` ให้ตรวจสอบว่า...

  • ในไฟล์ `server.js` (Backend) ได้ใส่ `app.use(cors());` ไว้หรือไม่
  • ได้ Restart Backend Server แล้วหรือยังหลังจากเพิ่มโค้ด

🧪 Test Scenarios (End-to-End)

ทดสอบการใช้งานจริงเหมือนเป็นผู้ใช้:

  1. เปิดหน้าเว็บ และดูว่ารายการร้านอาหารแสดงถูกต้อง
  2. ลองค้นหาและกรองข้อมูล ดูว่าผลลัพธ์ถูกต้องหรือไม่
  3. คลิกดูรายละเอียดร้าน และตรวจสอบว่าข้อมูลครบถ้วน
  4. ลองเขียนรีวิว แล้วกลับไปดูหน้ารายละเอียดอีกครั้งเพื่อดูว่ารีวิวใหม่และคะแนนเฉลี่ยอัปเดตหรือไม่
