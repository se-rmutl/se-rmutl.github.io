แล้ว React.js คืออะไร? 🤔

React คือ JavaScript library สำหรับสร้าง User Interfaces (UI) ที่สามารถจัดการสถานะ (state) และอัปเดตหน้าเว็บได้อย่างมีประสิทธิภาพ โดยไม่ต้องเขียนโค้ดจัดการ DOM เองให้วุ่นวาย

วิธีเก่า: จัดการ DOM ด้วย JS 😫

// เยอะ ซับซ้อน และผิดพลาดง่าย!
document.getElementById('username').innerHTML = 'สวัสดี ' + name;
document.getElementById('counter').innerHTML = count + ' ครั้ง';
document.getElementById('message').style.display = isVisible ? 'block' : 'none';

วิธี React: แค่บอกผลลัพธ์ที่ต้องการ 😊

// เขียนน้อย เข้าใจง่าย ผิดพลาดยาก!
function WelcomeComponent({ name, count, isVisible }) {
  return (
    <div>
      <h1>สวัสดี {name}</h1>
      <p>{count} ครั้ง</p>
      {isVisible && <div>ข้อความพิเศษ</div>}
    </div>
  );
}

เจาะลึกความแตกต่าง

วิธีเก่า (Imperative): เราต้องเขียนคำสั่ง "ทีละขั้นตอน" บอกเบราว์เซอร์ว่าต้องทำอะไรบ้าง (ไปหา id นี้, เปลี่ยนข้อความ, เปลี่ยนสไตล์) ซึ่งซับซ้อนและผิดพลาดง่ายเมื่อแอปใหญ่ขึ้น

วิธี React (Declarative): เราแค่ "ประกาศ" ว่าหน้าตา UI ควรเป็นอย่างไรเมื่อข้อมูล (state) เป็นแบบนี้ React จะจัดการส่วนที่เหลือให้เอง เราไม่ต้องยุ่งกับ DOM โดยตรงเลย

2/35

ประวัติ React.js 📚

  • 2011: Facebook เจอปัญหา UI ที่ซับซ้อนและอัปเดตได้ช้าใน News Feed
  • 2013: Jordan Walke วิศวกรของ Facebook สร้าง React และเปิดเป็น Open Source
  • ปัจจุบัน: ถูกใช้งานโดยบริษัทเทคโนโลยียักษ์ใหญ่ทั่วโลก เช่น Netflix, Uber, Airbnb, Instagram, และ WhatsApp

ทำไมถึงเป็นที่นิยม?

  • Declarative: แค่บอกว่า UI ควรหน้าตาเป็นอย่างไรในแต่ละ state แล้ว React จะจัดการอัปเดต DOM ให้เอง
  • Component-Based: สร้าง UI จากชิ้นส่วนเล็กๆ ที่เรียกว่า "Component" ซึ่งนำกลับมาใช้ใหม่ได้
  • Learn Once, Write Anywhere: เรียนรู้ครั้งเดียว แต่สามารถนำไปพัฒนาได้ทั้งเว็บ, แอปมือถือ (React Native), และแอปเดสก์ท็อป (Electron)
  • Virtual DOM: มีระบบจำลอง DOM ที่ช่วยให้อัปเดตหน้าเว็บได้เร็วและมีประสิทธิภาพสูงมาก
"ถ้า Facebook ใช้ React สำหรับผู้ใช้งาน 3 พันล้านคนได้ แสดงว่า... มันต้องเก่งจริงๆ! 💪"

React vs วิธีเก่า 🥊

เปรียบเทียบการสร้างปุ่มง่ายๆ ที่เมื่อคลิกแล้วจะแสดงข้อความ

วิธีเก่า (Vanilla JavaScript)

// ต้องจำเยอะ ทำเยอะ
const button = document.createElement('button');
button.textContent = 'คลิกฉัน';
button.addEventListener('click', function() {
    document.getElementById('output').innerHTML = 'ถูกคลิกแล้ว!';
});
document.body.appendChild(button);

วิธี React

// เข้าใจง่าย ทำน้อย
function MyButton() {
    const [clicked, setClicked] = useState(false);
    
    return (
        <div>
            <button onClick={() => setClicked(true)}>
                คลิกฉัน
            </button>
            {clicked && <p>ถูกคลิกแล้ว!</p>}
        </div>
    );
}

เจาะลึกโค้ด React

const [clicked, setClicked] = useState(false); คือการใช้ "Hook" เพื่อสร้าง state ชื่อ `clicked` ให้กับ component โดยมีค่าเริ่มต้นเป็น `false`

onClick={() => setClicked(true)} เมื่อปุ่มถูกคลิก เราจะเรียกฟังก์ชัน `setClicked` เพื่อเปลี่ยนค่า `clicked` ให้เป็น `true`

{clicked &&

...

} เป็นการทำ Conditional Rendering ครับ ถ้า `clicked` เป็น `true` ข้อความ `

` ก็จะแสดงผล

รู้จักกับ JSX 🎨 (JavaScript + XML)

JSX คือส่วนขยายของ JavaScript ที่ทำให้เราสามารถเขียนโค้ดที่หน้าตาเหมือน HTML ลงไปในไฟล์ JavaScript ได้เลย! มันคือ "Super Power" ⚡ ของ React

เขียนด้วย JSX (ง่ายกว่า)

const element = <h1>สวัสดี React!</h1>;

เบื้องหลัง (JS ธรรมดา)

// Babel จะแปลง JSX เป็นแบบนี้ให้เรา
const element2 = React.createElement('h1', null, 'สวัสดี React!');

ตัวอย่างการใช้ JavaScript ใน JSX

function Welcome() {
    const name = "นักศึกษา";
    const isLoggedIn = true;
    
    return (
        <div>
            <h1>สวัสดี {name}!</h1>
            {isLoggedIn ? (
                <p>ยินดีต้อนรับกลับมา</p>
            ) : (
                <p>กรุณาเข้าสู่ระบบ</p>
            )}
        </div>
    );
}

เบื้องหลังของ JSX

โค้ด JSX ที่เราเขียนไม่ได้ทำงานบนเบราว์เซอร์โดยตรงครับ ก่อนที่มันจะไปถึงเบราว์เซอร์ จะมีเครื่องมือที่ชื่อว่า Babel ทำการ "แปลง" โค้ด JSX ของเราให้กลายเป็นคำสั่ง JavaScript ธรรมดา (React.createElement(...)) ที่เบราว์เซอร์เข้าใจได้

ดังนั้น JSX จึงเป็นแค่ "Syntax Sugar" หรือรูปแบบการเขียนที่ทำให้เราในฐานะนักพัฒนา ทำงานได้ง่ายและอ่านโค้ดสะดวกขึ้นเท่านั้นเอง

กฎของ JSX ที่ต้องจำ 📋

1. ต้องมี Element ครอบนอกสุดเสมอ

ใช้ <div> หรือ <> (Fragment) ครอบ

// ✅ ถูก
return (
    <>
        <h1>หัวข้อ</h1>
        <p>เนื้อหา</p>
    </>
);

2. ใช้ className แทน class

เพราะ class เป็นคำสงวนใน JavaScript

// ❌ ผิด: <div class="container">
// ✅ ถูก
<div className="container">

3. ปิด Tag ให้ครบเสมอ

Tag ที่ไม่มี closing tag ใน HTML ต้องใส่ / ปิดท้าย

// ❌ ผิด: <img src="...">
// ✅ ถูก
<img src="image.jpg" />
<input type="text" />
<br />

4. Attributes ต้องเป็น camelCase

เช่น onclick ใน HTML จะกลายเป็น onClick

// ✅ ถูก
<button onClick={handleClick}>
<div tabIndex="-1">

JavaScript ใน JSX 💫

เราสามารถแทรกโค้ด JavaScript ใดๆ ก็ตามลงใน JSX ได้โดยใช้เครื่องหมายปีกกา {...}

function UserProfile() {
    const user = { name: "สมชาย", age: 20, hobbies: ["เขียนโค้ด", "เล่นเกม"] };
    
    return (
        <div className="profile">
            <h2>ชื่อ: {user.name}</h2>
            <p>อายุ: {user.age} ปี</p>
            <p>อีก {30 - user.age} ปี จะอายุ 30</p>
            <ul>
                {user.hobbies.map((hobby, index) => (
                    <li key={index}>{hobby}</li>
                ))}
            </ul>
            <button onClick={() => alert(`สวัสดี ${user.name}!`)}>
                ทักทาย
            </button>
        </div>
    );
}

การใช้ .map() ใน JSX

user.hobbies.map(...) คือการวนลูป array ที่ชื่อ `hobbies` ทีละตัว

ในแต่ละรอบ มันจะสร้าง JSX Element ใหม่ขึ้นมา (ในที่นี้คือ <li>) พร้อมกับข้อมูลจาก array (`hobby`)

ผลลัพธ์สุดท้ายคือ array ของ <li> Elements ซึ่ง React สามารถนำไปแสดงผลต่อได้ทันที เป็นวิธีที่นิยมที่สุดในการแสดงผลข้อมูลที่เป็นลิสต์ใน React ครับ

Interactive: Conditional Rendering 🤖

มาลองดู "การแสดงผลตามเงื่อนไข" แบบจริงๆ กันครับ ลองคลิกปุ่มเพื่อเปลี่ยนสถานะและดูผลลัพธ์ที่เปลี่ยนไป

📝 โค้ด Component
function StatusPanel({ isLoggedIn, hasMessage }) {
  return (
    <div>
      {/* 1. Ternary Operator */}
      {isLoggedIn ? (
        <p>✅ ยินดีต้อนรับ!</p>
      ) : (
        <p>❌ กรุณาเข้าสู่ระบบ</p>
      )}

      {/* 2. && Operator */}
      {hasMessage && (
        <div className="alert">
          🔔 คุณมีข้อความใหม่!
        </div>
      )}
    </div>
  );
}

อธิบาย Conditional Rendering 🤖

จากที่เราได้ลองเล่นไปเมื่อสักครู่ มันคือการแสดงผล Component หรือ Element ตามเงื่อนไข ซึ่งเป็นหัวใจสำคัญของ UI ที่ตอบสนองได้

1. ใช้ && Operator

ถ้าเงื่อนไขเป็นจริง ให้แสดง Element ด้านขวา (เหมาะกับกรณีที่ไม่มี else)

{hasMessage && <div className="alert">🔔 คุณมีข้อความใหม่!</div>}

2. ใช้ Ternary Operator ? :

เป็น if-else แบบย่อ เหมาะกับการสลับการแสดงผล 2 อย่าง

{isLoggedIn ? (<button>ออกจากระบบ</button>) : (<button>เข้าสู่ระบบ</button>)}

3. ใช้ If Statement (นอก JSX)

สำหรับเงื่อนไขที่ซับซ้อน สามารถใช้ if-else ปกติเพื่อกำหนดค่าตัวแปร แล้วนำไปแสดงผลใน JSX

function WeatherWidget({ weather }) {
    let weatherIcon;
    if (weather === 'sunny') weatherIcon = '☀️';
    else if (weather === 'rainy') weatherIcon = '🌧️';
    else weatherIcon = '☁️';
    
    return <div>{weatherIcon} {weather}</div>;
}

การวนลูป Lists ใน JSX 🔄

เราไม่สามารถใช้ for loop แบบปกติใน JSX ได้ แต่วิธีที่นิยมที่สุดคือการใช้ฟังก์ชัน .map() ของ Array เพื่อแปลงข้อมูลแต่ละตัวให้เป็น JSX Element

function StudentList() {
    const students = [
        { id: 1, name: "สมชาย", grade: "A" },
        { id: 2, name: "สมศรี", grade: "B+" },
        { id: 3, name: "สมหมาย", grade: "A-" }
    ];
    
    return (
        <ul>
            {students.map(student => (
                <li key={student.id}>
                    {student.name} - เกรด {student.grade}
                </li>
            ))}
        </ul>
    );
}

สิ่งสำคัญที่ต้องจำ!

  • key prop: ทุก Element ที่ถูกสร้างจากการวนลูป ต้องมี prop ชื่อ `key` ที่มีค่าไม่ซ้ำกัน (เช่น id จากฐานข้อมูล) เสมอ!

เจาะลึก `key` prop

ลองจินตนาการว่านักเรียนในห้องมีหลายคนชื่อ "สมชาย" ถ้าครูเรียกแค่ชื่อ อาจจะสับสนได้ แต่ถ้าครูเรียกด้วย "เลขประจำตัว" จะรู้ทันทีว่าคือคนไหน

key ก็เหมือน "เลขประจำตัว" ของ Element ในลิสต์ครับ React ใช้มันเพื่อติดตามว่า Element ไหนถูกเพิ่ม, ลบ, หรือสลับตำแหน่ง ทำให้การอัปเดตหน้าจอทำได้อย่างรวดเร็วและแม่นยำ โดยไม่ต้องสร้างลิสต์ใหม่ทั้งหมดทุกครั้งที่มีการเปลี่ยนแปลง

Event Handling ใน React 🎯

การจัดการ Event เช่น การคลิก (onClick) หรือการกรอกฟอร์ม (onChange) ใน React ทำได้ง่ายๆ โดยการส่งฟังก์ชันเข้าไปใน prop ของ Element

function InteractiveButton() {
    // Function สำหรับจัดการ Event
    const handleClick = () => {
        alert('ปุ่มถูกคลิก!');
    };
    
    const handleMouseOver = () => {
        console.log('เมาส์อยู่บนปุ่ม');
    };
    
    return (
        <button 
            onClick={handleClick}
            onMouseOver={handleMouseOver}
            onDoubleClick={() => alert('Double Click!')}
        >
            ลองคลิก!
        </button>
    );
}

Events ที่ใช้บ่อย:

  • onClick - คลิก
  • onChange - เปลี่ยนค่าในฟอร์ม (input, select)
  • onSubmit - ส่งฟอร์ม
  • onMouseOver / onMouseOut - เมาส์เข้า/ออก
  • onFocus / onBlur - โฟกัส/ออกจาก input

ทำไม React ถึงเร็ว? Virtual DOM 🚀

React ไม่ได้แก้ไข DOM จริงๆ (Real DOM) ทุกครั้งที่มีการเปลี่ยนแปลง ซึ่งเป็นกระบวนการที่ช้า แต่จะใช้สิ่งที่เรียกว่า Virtual DOM

ปัญหาของ DOM ธรรมดา 🐢

ทุกครั้งที่ .innerHTML เปลี่ยน เบราว์เซอร์ต้องคำนวณ Layout และวาดหน้าจอใหม่ทั้งหมด ทำให้ช้าและกระตุก

วิธีของ React - Virtual DOM ⚡

React จะเปรียบเทียบ Virtual DOM ของเก่ากับของใหม่ เพื่อหา "จุดที่แตกต่าง" แล้วไปอัปเดตแค่ส่วนนั้นใน Real DOM จริงๆ

ขั้นตอนการทำงาน:

  1. State ในแอปเปลี่ยนแปลง (เช่น ผู้ใช้คลิกปุ่ม)
  2. React สร้าง Virtual DOM snapshot ใหม่ขึ้นมาในหน่วยความจำ
  3. React เปรียบเทียบ snapshot ใหม่กับ snapshot เก่า (เรียกว่ากระบวนการ "Diffing")
  4. React คำนวณวิธีที่เร็วที่สุดในการอัปเดต Real DOM เพื่อให้เหมือนกับ snapshot ใหม่
  5. ทำการอัปเดต Real DOM เฉพาะจุดที่จำเป็น เท่านั้น

เปรียบเทียบ Virtual DOM

Real DOM: เหมือนการ "รีโนเวทบ้านทั้งหลัง" แม้จะเปลี่ยนแค่เฟอร์นิเจอร์ชิ้นเดียว ช้าและใช้พลังงานเยอะ

Virtual DOM: เหมือนการ "วางแผนบนพิมพ์เขียว" ก่อน React จะสร้างพิมพ์เขียว (Virtual DOM) ของเก่าและใหม่ขึ้นมา เปรียบเทียบกันว่ามีอะไรเปลี่ยนไปบ้าง แล้วจึงส่งคำสั่งให้ช่าง (เบราว์เซอร์) ไปแก้ไข "เฉพาะจุดนั้น" ในบ้านจริง (Real DOM) ซึ่งเร็วกว่าและมีประสิทธิภาพกว่ามาก

การสร้าง Component แรก 👶

Component คือหัวใจของ React มันคือชิ้นส่วน UI ที่นำกลับมาใช้ซ้ำได้ พูดง่ายๆ มันคือฟังก์ชัน JavaScript ที่ return ค่าเป็น JSX

Component ง่ายๆ:

// นี่คือ Component ชื่อ WelcomeCard
function WelcomeCard() {
    return (
        <div className="welcome-card">
            <h2>🎉 ยินดีต้อนรับ!</h2>
            <p>สวัสดีนักพัฒนาเว็บมือใหม่</p>
        </div>
    );
}

การนำ Component ไปใช้งาน:

เราสามารถเรียกใช้ Component เหมือนเป็น HTML Tag ที่เราสร้างขึ้นเอง

function App() {
    return (
        <div>
            <WelcomeCard />
            <WelcomeCard />
            <WelcomeCard />
        </div>
    );
}

กฎการตั้งชื่อ Component: ✅ ต้องขึ้นต้นด้วยตัวพิมพ์ใหญ่เสมอ (PascalCase) เช่น UserProfile, ProductCard

Component ที่มี Props 🎁

Props (ย่อมาจาก Properties) คือวิธีการส่งข้อมูลจาก Component แม่ (Parent) ไปยัง Component ลูก (Child) ทำให้เราสามารถสร้าง Component ที่ยืดหยุ่นและนำไปใช้ซ้ำกับข้อมูลที่แตกต่างกันได้

Component ที่รับ Props:

// สังเกต { name, major, gpa } ใน parameter คือการรับ props
function StudentCard({ name, major, gpa }) {
    return (
        <div className="student-card">
            <h3>{name}</h3>
            <p>สาขา: {major}</p>
            <p>GPA: {gpa.toFixed(2)} {gpa >= 3.5 && '🌟'}</p>
        </div>
    );
}

การส่ง Props ตอนใช้งาน:

function ClassRoom() {
    return (
        <div>
            <StudentCard name="สมชาย" major="วิศวกรรมซอฟต์แวร์" gpa={3.75} />
            <StudentCard name="สมศรี" major="วิทยาการคอมพิวเตอร์" gpa={3.25} />
        </div>
    );
}

Props เปรียบเสมือน...

...อาร์กิวเมนต์ (arguments) ของฟังก์ชันครับ Component คือฟังก์ชัน และ Props คือข้อมูลที่เราส่งเข้าไปเพื่อให้ฟังก์ชันนั้นทำงานต่างกันได้

ในตัวอย่าง StudentCard เปรียบเสมือน "แม่แบบบัตรนักศึกษา" ส่วน props ที่เราส่งเข้าไป (name, major, gpa) คือ "ข้อมูลเฉพาะ" ของนักศึกษาแต่ละคน ทำให้เราใช้แม่แบบเดียวสร้างบัตรนักศึกษาหลายใบที่มีข้อมูลต่างกันได้

Default Props และ PropTypes 🛡️

เราสามารถทำให้ Component ของเราแข็งแกร่งและป้องกันข้อผิดพลาดได้ดียิ่งขึ้น

1. Default Props

กำหนดค่าเริ่มต้นให้กับ props ในกรณีที่ไม่ได้ถูกส่งมา

function ProductCard({ name, price = 0, inStock = true }) {
    return (
        <div>
          <h3>{name}</h3>
          <p>{price > 0 ? `฿${price}` : 'ฟรี!'}</p>
          {!inStock && <div>สินค้าหมด</div>}
        </div>
    );
}
// การใช้งาน: <ProductCard name="แอปฟรี" /> // price และ inStock จะใช้ค่า default

2. PropTypes (Optional)

ใช้ไลบรารี่ prop-types เพื่อตรวจสอบประเภทของข้อมูลที่ส่งมา และแสดงคำเตือนใน Console หากข้อมูลผิดประเภท

import PropTypes from 'prop-types';

ProductCard.propTypes = {
    name: PropTypes.string.isRequired,
    price: PropTypes.number,
    inStock: PropTypes.bool
};

ประโยชน์:

  • ป้องกันข้อผิดพลาด: ตรวจสอบข้อมูลที่ส่งมาให้ Component
  • ง่ายต่อการ Debug: แจ้งเตือนเมื่อ props ผิดประเภท
  • เป็นเอกสารในตัว: อ่านแล้วรู้ทันทีว่า Component นี้ต้องการ props อะไรบ้าง

Component Composition 🧩

คือหลักการ "Single Responsibility Principle" ใน React โดยการสร้าง Component เล็กๆ ที่ทำหน้าที่เฉพาะอย่าง แล้วนำมา "ประกอบร่าง" กันเป็น Component ที่ใหญ่และซับซ้อนขึ้น

ตัวอย่าง: การสร้าง UserCard

1. สร้าง Components เล็กๆ:

function Avatar({ user }) { ... }
function UserInfo({ user }) { ... }
function UserActions() { ... }

แต่ละอันทำหน้าที่เดียว: แสดงรูป, แสดงข้อมูล, แสดงปุ่ม

2. นำมาประกอบกัน:

function UserCard({ user }) {
    return (
        <div className="user-card">
            <Avatar user={user} />
            <UserInfo user={user} />
            <UserActions />
        </div>
    );
}

UserCard ไม่ได้ทำอะไรมากนอกจากเป็น "ภาชนะ" บรรจุ Component อื่นๆ

ข้อดีของ Composition:

  • แบ่งแยก (Separation of Concerns): แต่ละ Component ทำหน้าที่เดียว ทำให้โค้ดสะอาด
  • นำกลับมาใช้ได้ (Reusability): สามารถนำ Avatar ไปใช้ในที่อื่นๆ ได้โดยไม่ต้องแก้โค้ด
  • แก้ไขง่าย (Maintainability): อยากเปลี่ยนแค่ปุ่ม ก็ไปแก้ที่ UserActions ที่เดียว ไม่กระทบส่วนอื่น

Quick Quiz 🧠

ทดสอบความเข้าใจ

ทำไมเราถึงต้องใช้ key prop ที่มีค่าไม่ซ้ำกัน เวลาที่เราทำการวนลูปสร้าง Element ใน React?

คำตอบ: เพื่อประสิทธิภาพในการอัปเดต (Efficient Updates)

React ใช้ `key` เพื่อเป็น "บัตรประจำตัว" ของแต่ละ Element ในลิสต์ เมื่อข้อมูลมีการเปลี่ยนแปลง (เช่น เพิ่ม, ลบ, หรือสลับตำแหน่ง) React จะดูที่ `key` เพื่อ:

  • ระบุตัวตน: รู้ว่า element ไหนคืออันไหน ไม่ต้องสร้างใหม่ทั้งหมด
  • ลดการทำงาน: อัปเดตเฉพาะ DOM ของ element ที่เปลี่ยนแปลงจริงๆ
  • ป้องกันบั๊ก: โดยเฉพาะกับ component ที่มี state ของตัวเอง

สรุป: การใส่ `key` ช่วยให้ React ทำงานได้อย่างชาญฉลาดและรวดเร็วขึ้นมาก!

คำถามสำคัญ ❓

1. JSX ต่างจาก HTML อย่างไร?

JSX สามารถเขียน JavaScript expression ข้างในโดยใช้ {...} ได้, ใช้ className แทน class, และ attributes เป็นแบบ camelCase (onClick)

2. Component ต่างจาก function ธรรมดาอย่างไร?

Component return ค่าเป็น JSX, ชื่อต้องขึ้นต้นด้วยตัวพิมพ์ใหญ่ (PascalCase), และสามารถถูกเรียกใช้งานในรูปแบบของ JSX Tag (<MyComponent />) ได้

3. Props ส่งได้แค่ string หรือไม่?

ไม่, Props สามารถส่งข้อมูลได้ทุกประเภทใน JavaScript: number, boolean, object, array, หรือแม้กระทั่ง function!

4. ทำไมเราถึงควรใช้ Component Composition?

เพื่อทำให้โค้ดสะอาดขึ้น, นำกลับมาใช้ซ้ำได้ง่าย และแก้ไขได้สะดวกโดยไม่กระทบส่วนอื่นๆ (Single Responsibility Principle)

`children` props ทำงานอย่างไร?

children เป็น prop พิเศษที่ React สร้างให้เองอัตโนมัติ มันคือทุกอย่างที่เราใส่ไว้ "ระหว่าง" tag เปิดและปิดของ Component

ในตัวอย่าง <Card>...</Card> ทุกอย่างที่อยู่ตรง `...` จะถูกส่งไปใน prop ที่ชื่อ `children` ทำให้ Component Card ของเรากลายเป็น "กล่อง" หรือ "กรอบ" ที่สามารถนำเนื้อหาอะไรก็ได้ไปใส่ ทำให้ยืดหยุ่นและนำไปใช้ซ้ำได้ง่ายมากครับ

สรุป: ปุ่ม "แก้ไข" ในหน้าจอนี้จึงเป็นแค่การแสดงผลตัวอย่างว่าเราสามารถใส่ปุ่มเข้าไปใน Card ได้ ไม่ได้ถูกทำให้ทำงานจริงในสไลด์นี้ครับ

CSS ใน React - Styling Components 🎨

มีหลายวิธีในการใส่สไตล์ให้กับ Component ใน React แต่นี่คือ 3 วิธีที่นิยมที่สุด:

1. CSS ธรรมดา (External CSS File) - แนะนำสำหรับผู้เริ่มต้น

สร้างไฟล์ .css (เช่น App.css) แล้ว import เข้ามาในไฟล์ Component ของคุณ

/* App.css */
.profile-card { border: 1px solid #ddd; padding: 20px; }
// App.js
import './App.css'; // Import ไฟล์ CSS
function App() { return <div className="profile-card">...</div> }

2. Inline Styles

เขียน CSS เป็น JavaScript object แล้วส่งเข้าไปใน style prop (property ต้องเป็น camelCase)

function StyledButton() {
    const buttonStyle = {
        backgroundColor: '#007bff',
        color: 'white',
        borderRadius: '5px'
    };
    return <button style={buttonStyle}>คลิก</button>
}

3. CSS Modules (ขั้นสูง)

ตั้งชื่อไฟล์เป็น .module.css React จะสร้าง class name ที่ไม่ซ้ำกันให้โดยอัตโนมัติ ป้องกันปัญหา class name ชนกัน

// Button.js
import styles from './Button.module.css';
function Button() { return <button className={styles.primary}>คลิก</button> }

การจัดการ Arrays ใน React 📊

เราสามารถใช้ Array methods ของ JavaScript (.map, .filter, .reduce) เพื่อจัดการข้อมูลและแสดงผลแบบไดนามิกได้อย่างง่ายดาย

function StudentGrades() {
    const grades = [
        { subject: 'คณิตศาสตร์', score: 85 }, { subject: 'ฟิสิกส์', score: 78 },
        { subject: 'เคมี', score: 92 }, { subject: 'อังกฤษ', score: 65 }
    ];
    
    // กรองข้อมูล (filter)
    const excellentGrades = grades.filter(g => g.score >= 85);
    
    // คำนวณค่ารวม (reduce)
    const totalScore = grades.reduce((sum, grade) => sum + grade.score, 0);
    const averageScore = totalScore / grades.length;
    
    return (
        <div>
            <h2>คะแนนเฉลี่ย: {averageScore.toFixed(2)}</h2>
            
            <h3>🌟 วิชาที่ได้คะแนนดีเยี่ยม:</h3>
            <ul>{excellentGrades.map(g => <li key={g.subject}>{g.subject}</li>)}</ul>
            
            <h3>📚 รายการคะแนนทั้งหมด:</h3>
            <ul>
                {grades.map(grade => (
                    <li key={grade.subject} className={grade.score < 70 ? 'warning' : ''}>
                        {grade.subject}: {grade.score} คะแนน 
                    </li>
                ))}
            </ul>
        </div>
    );
}

ทำความเข้าใจ React Ecosystem 🌍

React เป็นแค่ Library สำหรับสร้าง UI แต่ในโลกความเป็นจริง เราต้องใช้เครื่องมือและไลบรารี่อื่นๆ อีกมากมาย นี่คือส่วนหนึ่งของ Ecosystem ที่ควรรู้จัก:

เครื่องมือพัฒนา (Build Tools / Frameworks)

  • Create React App / Vite: เครื่องมือสำหรับสร้างและตั้งค่าโปรเจกต์ React ได้อย่างรวดเร็ว (ที่เราใช้อยู่คือ Create React App)
  • Next.js / Remix: Framework ที่สร้างบน React เพิ่มฟีเจอร์สำคัญๆ เช่น Server-Side Rendering, Routing, etc.

Libraries ที่ใช้บ่อย

  • UI Libraries: Material-UI (MUI), Ant Design, Chakra UI (ชุด Components สำเร็จรูปสวยๆ)
  • Routing: React Router (สำหรับทำหน้าเว็บหลายหน้า หรือ Single Page Application)
  • State Management: Redux, Zustand, Recoil (สำหรับจัดการ State ที่ซับซ้อนในแอปขนาดใหญ่)
  • Data Fetching: React Query, SWR, Axios (สำหรับดึงข้อมูลจาก API)

คำแนะนำ: เริ่มจากพื้นฐานของ React ให้แข็งแรงก่อน แล้วค่อยๆ เรียนรู้เครื่องมือเหล่านี้เพิ่มเติมเมื่อจำเป็น

React Fundamentals:

  • React คืออะไร: Library สำหรับสร้าง UI ที่ทำงานด้วย Component
  • Virtual DOM: กลไกที่ทำให้ React ทำงานได้รวดเร็ว
  • Component: ชิ้นส่วน UI ที่นำกลับมาใช้ซ้ำได้ (ฟังก์ชันที่ return JSX)
  • Props: วิธีการส่งข้อมูลจาก Component แม่ไปลูก

JSX Mastery:

  • JSX Syntax: เขียน HTML ใน JavaScript
  • JavaScript ใน JSX: ใช้ {...} เพื่อแทรกโค้ด JS
  • Conditional Rendering: แสดงผลตามเงื่อนไข (&&, ? :)
  • Lists & Keys: แสดงข้อมูล Array ด้วย .map() และ key
  • Event Handling: จัดการ Events เช่น onClick

เมื่อเช้า: รู้จักแค่ HTML + JavaScript

คำถามที่พบบ่อย (FAQ) 🙋

  • React ยากไหม?
    A: ช่วงแรกอาจจะรู้สึกแตกต่างจากการเขียนเว็บแบบเดิม แต่เมื่อเข้าใจแนวคิดเรื่อง Component, Props, และ State แล้ว จะพบว่ามันทำให้การพัฒนาเว็บขนาดใหญ่เป็นเรื่องง่ายและสนุกขึ้นมาก!
  • จำเป็นต้องรู้ JavaScript เก่งมาก่อนไหม?
    A: รู้พื้นฐานก็เริ่มได้ แต่ยิ่งเข้าใจ JavaScript ลึกซึ้ง (โดยเฉพาะ ES6+) ก็จะยิ่งเขียน React ได้ดีและมีประสิทธิภาพมากขึ้น
  • หลังเรียนจบคอร์สนี้จะหางานได้ไหม?
    A: React Developer เป็นหนึ่งในตำแหน่งงานที่เป็นที่ต้องการมากที่สุดในวงการพัฒนาซอฟต์แวร์ทั่วโลก การมีพื้นฐานที่แข็งแรงจะเปิดโอกาสให้คุณอย่างมหาศาล

เตรียมตัวสำหรับหัวข้อถัดไป 🚀

หัวข้อสำคัญ: State & Hooks

ถึงตอนนี้ Component ของเราสามารถรับข้อมูลผ่าน Props และแสดงผลได้ แต่ยังไม่สามารถ "เปลี่ยนแปลง" หรือ "มีชีวิต" ได้ด้วยตัวเอง

ในหัวข้อถัดไป เราจะเรียนรู้เกี่ยวกับ useState ซึ่งเป็นเครื่องมือที่ทรงพลังที่สุดในการ:

  • ทำให้ Component "จำ" ข้อมูลของตัวเองได้ (State)
  • ทำให้ Component re-render หรือ "อัปเดตหน้าจอ" ใหม่ เมื่อข้อมูลนั้นเปลี่ยนไป
  • สร้าง User Interface ที่ interactive อย่างแท้จริง เช่น การนับคะแนน, การเปิด-ปิดเมนู, หรือการกรอกฟอร์ม
State คือสิ่งที่ทำให้แอปพลิเคชัน React ของเรามีชีวิตชีวา!
