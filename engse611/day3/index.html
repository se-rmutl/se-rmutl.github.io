React & JSX

ปัญหาคืออะไร?

การจัดการ DOM ด้วย JavaScript ธรรมดานั้นซับซ้อนและช้าเมื่อ UI มีความซับซ้อนสูง

ทางออกคืออะไร?

React ให้เราสร้าง UI แบบ "Declarative" (บอกผลลัพธ์) ด้วย JSX ซึ่งทำให้โค้ดอ่านง่ายและเร็วขึ้นด้วย Virtual DOM

ประยุกต์ใช้ต่อไป?

เราจะใช้ JSX เป็นภาษาหลักในการสร้าง "Component" ซึ่งเป็นชิ้นส่วน UI ทั้งหมดของแอปพลิเคชัน

2. Components & Props

การสร้าง UI แบบตัวต่อ LEGO

Components & Props

ปัญหาคืออะไร?

การสร้าง UI ทั้งหน้าในไฟล์เดียวทำให้โค้ดซ้ำซ้อน, ดูแลรักษายาก, และนำกลับมาใช้ใหม่ไม่ได้

ทางออกคืออะไร?

เราแบ่ง UI เป็นชิ้นส่วนเล็กๆ (Component) ที่ทำงานแยกกัน และส่งข้อมูลระหว่างกันผ่าน "ท่อ" ที่เรียกว่า Props

ประยุกต์ใช้ต่อไป?

เราจะสร้างแอปพลิเคชันโดยการประกอบ Component หลายๆ ชิ้นเข้าด้วยกัน และทำให้มันมีชีวิตชีวาด้วย "State"

Infographic: การสร้าง UI ด้วย Component

เหมือนนำชิ้นส่วน LEGO (Component) มาประกอบกันเป็นบ้าน (Application)

3. State Management

การทำให้ Component "มีชีวิต" และ "ความจำ"

Infographic: การเปลี่ยนแปลงของ UI

State คือสิ่งที่ทำให้ UI เปลี่ยนแปลงได้เมื่อมี Interaction เกิดขึ้น

State & useState

ปัญหาคืออะไร?

Component ไม่สามารถ "จดจำ" ข้อมูลที่เปลี่ยนแปลงได้ด้วยตัวเอง (เช่น จำนวนครั้งที่คลิก, ข้อความที่พิมพ์)

ทางออกคืออะไร?

เราใช้ `useState` เพื่อสร้าง "หน่วยความจำ" (State) ให้กับ Component ทำให้มันสามารถเก็บและอัปเดตข้อมูลได้เมื่อผู้ใช้โต้ตอบ

ประยุกต์ใช้ต่อไป?

เราจะนำ State มาใช้สร้างเกม, จัดการฟอร์ม, และสร้าง UI ที่ Interactive ทุกรูปแบบใน Lab สุดท้ายของวันนี้
