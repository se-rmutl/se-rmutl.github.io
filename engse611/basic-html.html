1/30

HTML คืออะไร?

HTML = HyperText Markup Language
เปรียบเสมือน พิมพ์เขียว (Blueprint) ของทุกหน้าเว็บบนโลกอินเทอร์เน็ต เป็นภาษาหลักที่เบราว์เซอร์ใช้อ่านเพื่อสร้างโครงสร้างและแสดงเนื้อหาต่างๆ

เปรียบเทียบกับการสร้างบ้าน

ถ้าเว็บไซต์คือบ้านหนึ่งหลัง HTML ก็คือส่วนประกอบโครงสร้างทั้งหมด:

  • <h1>, <h2>: คือ เสาและคาน ที่เป็นโครงหลัก
  • <p>: คือ ผนังห้องต่างๆ ที่แบ่งพื้นที่เนื้อหา
  • <img>: คือ หน้าต่างและประตู
  • <a>: คือ ทางเดินเชื่อมระหว่างห้อง
2/30

ทำไมต้อง HTML5?

HTML5 คือมาตรฐานปัจจุบันที่ทุกเว็บไซต์สมัยใหม่ใช้ มีการปรับปรุงจากเวอร์ชันเก่ามากมาย:

HTML เก่า HTML5
⚠️ ซับซ้อน✅ เรียบง่ายและมีความหมาย (Semantic)
❌ ไม่รองรับมือถือโดยตรง✅ ออกแบบมาเพื่อทุกอุปกรณ์ (Mobile-First)
🐌 ช้ากว่า⚡ ประสิทธิภาพดีกว่า
📱 ฟีเจอร์จำกัด🚀 มีฟีเจอร์ใหม่ๆ มากมาย (Video, Audio, Canvas)

HTML5 คือรากฐานที่แข็งแรงสำหรับการพัฒนาเว็บในปัจจุบัน!

4/30

โครงสร้างพื้นฐานของไฟล์ HTML

ทุกไฟล์ HTML จะมีโครงสร้างหลักที่เหมือนกันเสมอ ซึ่งประกอบไปด้วยส่วนต่างๆ ที่จำเป็นสำหรับเบราว์เซอร์

<!DOCTYPE html>
<html lang="th">
<head>
    <!-- ส่วนข้อมูลเบื้องหลังของเว็บ -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>หน้าแรกของฉัน</title>
</head>
<body>
    <!-- ส่วนเนื้อหาที่จะแสดงผลให้ผู้ใช้เห็น -->
    <h1>สวัสดีชาวโลก!</h1>
    <p>นี่คือเว็บไซต์แรกของฉัน</p>
</body>
</html>

Tip: ใน VS Code พิมพ์ ! แล้วกด Tab เพื่อสร้างโครงสร้างนี้อัตโนมัติ!

5/30

แยกชิ้นส่วนโครงสร้าง HTML

1. DOCTYPE Declaration

<!DOCTYPE html>

เป็นการประกาศเพื่อบอกเบราว์เซอร์ว่าเอกสารนี้เป็นมาตรฐาน HTML5

2. HTML Element

<html lang="th">...</html>

เป็น Tag ตัวหลักที่ห่อหุ้มทุกอย่างในหน้านี้ไว้ lang="th" คือการบอกว่าเนื้อหาหลักเป็นภาษาไทย

3. Head & Body Section

  • <head>: ส่วนที่เก็บข้อมูลเกี่ยวกับเว็บไซต์ เช่น ชื่อเว็บ, การเข้ารหัสอักขระ จะไม่แสดงผลบนหน้าเว็บโดยตรง
  • <body>: ส่วนของเนื้อหาทั้งหมดที่ต้องการให้ผู้ใช้มองเห็น
6/30

เจาะลึกส่วน Head

ภายใน <head> จะมี Tags ที่สำคัญสำหรับการตั้งค่าหน้าเว็บ:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>หน้าแรกของฉัน</title>
</head>
  • <meta charset="UTF-8">: กำหนดชุดอักขระเป็น UTF-8 เพื่อให้รองรับภาษาไทย
  • <meta name="viewport" ...>: ตั้งค่าการแสดงผลสำหรับอุปกรณ์พกพา ทำให้เว็บแสดงผลได้ดีบนมือถือ (Responsive)
  • <title>...</title>: กำหนดชื่อที่แสดงบน Tab ของเบราว์เซอร์
7/30

สำรวจส่วน Body

<body> คือส่วนที่สำคัญที่สุด เพราะเป็นที่อยู่ของเนื้อหาทั้งหมดที่เราต้องการให้ผู้ใช้มองเห็น

<body>
    <h1>สวัสดีชาวโลก!</h1>
    <p>นี่คือเว็บไซต์แรกของฉัน</p>
</body>

ในส่วน Body สามารถใส่ได้ทุกอย่าง เช่น:

  • ข้อความ, หัวข้อ
  • รูปภาพ, วิดีโอ
  • ลิงก์, ปุ่ม
  • ตาราง, ฟอร์ม

คำถามทดสอบความเข้าใจ

ถ้าคุณต้องการเปลี่ยนชื่อเว็บไซต์ที่แสดงบน Tab ของเบราว์เซอร์ คุณจะแก้ไขโค้ดใน Tag ใด?
A) <body>
B) <title>
C) <h1>
D) <meta>

คำตอบคือ B) <title>

เพราะ <title> ถูกออกแบบมาเพื่อกำหนดชื่อของเอกสารโดยเฉพาะ และต้องอยู่ภายใน <head> เสมอ

9/30

HTML Tags คืออะไร?

Tag คือ "คำสั่ง" หรือ "เครื่องหมาย" ที่เราใช้เพื่อบอกเบราว์เซอร์ว่าเนื้อหาส่วนนั้นๆ คืออะไรและควรแสดงผลอย่างไร

รูปแบบพื้นฐาน:

<tagname>เนื้อหาที่ต้องการจัดรูปแบบ</tagname>
  • <tagname> คือ Opening Tag (แท็กเปิด)
  • </tagname> คือ Closing Tag (แท็กปิด) ซึ่งจะมีเครื่องหมาย / นำหน้า

กฎสำคัญ:

  • เปิด Tag แล้วต้องปิด Tag เสมอ (ยกเว้นบาง Tag เช่น <img>, <br>)
  • ควรเขียนชื่อ Tag ด้วยตัวพิมพ์เล็กทั้งหมดเพื่อความเป็นมาตรฐาน

Tags สำหรับรายการ (Lists)

ใช้สำหรับจัดกลุ่มข้อมูลให้อ่านง่าย มี 2 ประเภทหลัก

1. Unordered List <ul>

รายการแบบไม่มีลำดับ (ใช้สัญลักษณ์นำหน้า)

<ul>
    <li>กาแฟ</li>
    <li>ชา</li>
</ul>

2. Ordered List <ol>

รายการแบบมีลำดับ (ใช้ตัวเลขนำหน้า)

<ol>
    <li>ตื่นนอน</li>
    <li>แปรงฟัน</li>
</ol>

ทั้งสองแบบจะใช้ <li> (List Item) สำหรับแต่ละรายการข้างใน

HTML Attributes คืออะไร?

Attributes คือ "ข้อมูลเพิ่มเติม" ที่เราใส่เข้าไปใน Opening Tag เพื่อกำหนดคุณสมบัติพิเศษหรือพฤติกรรมของ Tag นั้นๆ

รูปแบบ: <tagname attribute="value">

<!-- href และ target คือ attributes ของ a tag -->
<a href="https://google.com" target="_blank">Google</a>

<!-- src, alt, width คือ attributes ของ img tag -->
<img src="photo.jpg" alt="รูปภาพ" width="200">

<!-- id และ class คือ attributes ที่ใช้บ่อยสำหรับ CSS และ JavaScript -->
<p id="intro" class="highlight-text">ข้อความแนะนำ</p>
19/30

โครงสร้างเว็บที่มีความหมาย (Semantic HTML)

แทนที่จะใช้ <div> สำหรับทุกอย่าง, HTML5 มี Tag ที่มีความหมายในตัวเองเพื่อช่วยจัดโครงสร้างให้ชัดเจน

<header>: ส่วนหัวเว็บ

<nav>: เมนู

<main>: เนื้อหาหลัก

<footer>: ส่วนท้ายเว็บ

ประโยชน์: โค้ดอ่านง่าย, ดีต่อ SEO, และดีต่อผู้ใช้งานโปรแกรมอ่านจอ (Screen Reader)

20/30

Semantic Tags ที่ควรรู้จัก

การเลือกใช้ Tag ที่ถูกความหมาย ทำให้โค้ดของเราเป็นมืออาชีพมากขึ้น

Tag ใหม่ (Semantic) ความหมายและการใช้งาน
<header>ส่วนหัวของเว็บหรือ Section (เช่น โลโก้, ชื่อเว็บ)
<nav>กลุ่มของลิงก์เมนูนำทางหลัก
<main>เนื้อหาหลักเพียงหนึ่งเดียวของหน้านั้นๆ
<article>เนื้อหาที่สมบูรณ์ในตัวเอง เช่น บทความ, ข่าว, โพสต์ใน blog
<section>กลุ่มของเนื้อหาที่เกี่ยวข้องกัน มีหัวข้อของตัวเอง
<aside>เนื้อหาข้างเคียงที่ไม่ใช่ส่วนหลัก เช่น sidebar, โฆษณา
<footer>ส่วนท้ายของเว็บหรือ Section (เช่น copyright, ข้อมูลติดต่อ)

Comments ใน HTML

คือข้อความที่เราเขียนไว้ในโค้ด แต่จะไม่แสดงผลบนหน้าเว็บไซต์ ใช้สำหรับจดบันทึกหรืออธิบายโค้ด

รูปแบบการเขียน:

<!-- นี่คือ comment หนึ่งบรรทัด -->

<h1>หัวข้อ</h1> <!-- TODO: เปลี่ยนเป็นหัวข้อที่ดีกว่านี้ -->

<!-- 
    นี่คือ comment แบบหลายบรรทัด
    สามารถใช้อธิบายโค้ดที่ซับซ้อนได้
-->

ประโยชน์:

  • จดบันทึกช่วยจำสำหรับตัวเอง
  • อธิบายโค้ดที่ซับซ้อนให้เพื่อนร่วมทีมเข้าใจ
  • ปิดการใช้งานโค้ดบางส่วนชั่วคราว (Comment out)

HTML Validation

คือการตรวจสอบว่าโค้ด HTML ของเราเขียนได้ถูกต้องตามมาตรฐานหรือไม่

HTML ที่ดี:

<h1>หัวข้อ</h1>
<p>ย่อหน้า</p>
<img src="img.jpg" alt="รูป">

HTML ที่ผิด:

<h1>หัวข้อ
<p>ย่อหน้า</P> <!-- P ตัวใหญ่ -->
<img src="img.jpg"> <!-- ไม่มี alt -->

เครื่องมือ: สามารถใช้ W3C Markup Validator เพื่อตรวจสอบโค้ดของเราได้

Best Practices

เคล็ดลับการเขียน HTML ที่ดี

ใช้ Semantic Tags

ใช้ <header>, <main>, <footer> แทน <div> เพื่อทำให้โค้ดมีความหมาย

เพิ่ม Alt ให้รูปเสมอ

alt attribute สำคัญต่อ Accessibility และ SEO

ใช้ Title ที่สื่อความหมาย

เช่น "ร้านอาหารไทยแสนอร่อย - หน้าแรก" ดีกว่าแค่ "หน้าแรก"

เขียน HTML ที่อ่านง่าย

ใช้การย่อหน้า (Indentation) อย่างสม่ำเสมอ เพื่อให้โค้ดเป็นระเบียบและง่ายต่อการแก้ไข

การบ้าน (Lab)

สร้างเว็บไซต์แนะนำตัวเองด้วย HTML

ใช้ความรู้ทั้งหมดที่เรียนในวันนี้ สร้างไฟล์ index.html หนึ่งไฟล์ ที่เป็นหน้าเว็บแนะนำตัวเอง โดยต้องมีองค์ประกอบดังนี้:

  • <h1> ที่เป็นชื่อของคุณ
  • รูปภาพของคุณ (ใช้ placeholder ก็ได้) พร้อม alt attribute
  • <h2> สำหรับหัวข้อต่างๆ เช่น "ประวัติส่วนตัว", "งานอดิเรก", "เป้าหมาย"
  • <p> ที่มีเนื้อหาอธิบายในแต่ละหัวข้อ
  • <ul> หรือ <ol> สำหรับแสดงรายการ (เช่น รายการงานอดิเรก)
  • <a> ที่เป็นลิงก์ไปยัง Social Media ของคุณ (เปิดในแท็บใหม่)
  • ใช้ Semantic Tags (header, main, footer) ในการจัดโครงสร้าง

เป้าหมาย: เพื่อทบทวนการใช้ Tags ต่างๆ ที่ได้เรียนมาในสถานการณ์จริง
