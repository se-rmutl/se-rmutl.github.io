โลกสองใบ: Client vs. Server

การพัฒนาเว็บสมัยใหม่แบ่งออกเป็น 2 ส่วนหลักๆ ที่ทำงานร่วมกันแต่มีหน้าที่แตกต่างกันอย่างสิ้นเชิง

Client-Side (ฝั่งผู้ใช้)

  • ที่อยู่: ทำงานบนเว็บเบราว์เซอร์ของผู้ใช้
  • หน้าที่: แสดงผลหน้าตาเว็บ (UI) และตอบสนองต่อผู้ใช้
  • หน่วยความจำ: ชั่วคราว (State) ข้อมูลหายเมื่อปิดเว็บ
  • ข้อจำกัด: เข้าถึงฐานข้อมูลโดยตรงไม่ได้

Server-Side (ฝั่งเรา)

  • ที่อยู่: ทำงานบนคอมพิวเตอร์เซิร์ฟเวอร์
  • หน้าที่: จัดการ Logic และเป็นแหล่งข้อมูลจริง (Source of Truth)
  • หน่วยความจำ: ถาวร เชื่อมต่อฐานข้อมูล/ไฟล์ได้
  • พลังพิเศษ: เข้าถึงไฟล์, ฐานข้อมูล, ระบบอื่นๆ ได้

สะพานเชื่อม: กลไกการทำงานของ API

เพื่อให้ทั้งสองฝั่งสื่อสารกันได้ เราต้องสร้าง "สะพาน" ที่เรียกว่า **API (Application Programming Interface)** โดยใช้ HTTP เป็นช่องทางในการส่ง "คำขอ" (Request) และ "คำตอบ" (Response)

1. React App ส่งคำขอ (Request)

เมื่อผู้ใช้คลิก, React จะใช้ fetch() ส่งคำขอ HTTP ไปยัง URL ของ Backend

2. Express Server รับคำขอ

เซิร์ฟเวอร์ Node.js ของเราดักฟัง Request และ Express Router จะจับคู่ URL ที่เข้ามา

3. Server ประมวลผล

Controller Function ทำงานตาม Logic เช่น อ่านข้อมูลจากไฟล์ foods.json

4. Server ส่งคำตอบ (Response)

เซิร์ฟเวอร์ส่งข้อมูลกลับไปในรูปแบบ JSON พร้อมกับ Status Code (เช่น 200 OK)

5. React App อัปเดตหน้าเว็บ

React รับข้อมูล JSON มาอัปเดต State ทำให้หน้าเว็บ (UI) เปลี่ยนแปลงอัตโนมัติ

ข้อตกลงในการสื่อสาร: Frontend vs. Backend

เพื่อให้การเชื่อมต่อราบรื่น ทั้งสองฝั่งต้องมี "ข้อตกลง" หรือหน้าที่ความรับผิดชอบของตัวเอง

หน้าที่ของ Frontend

  • ส่งคำขอ (Request): เรียก API Endpoint ที่ถูกต้อง พร้อมส่งข้อมูลตามรูปแบบที่กำหนด
  • รับคำตอบ (Response): รอรับข้อมูล JSON และจัดการข้อมูลที่ได้มา
  • จัดการสถานะ UI: แสดงสถานะ "กำลังโหลด" ระหว่างรอ และอัปเดตหน้าเว็บเมื่อได้รับข้อมูล
  • จัดการข้อผิดพลาด: เตรียมพร้อมรับมือเมื่อ Backend ส่ง Error กลับมา และแสดงข้อความให้ผู้ใช้ทราบ

หน้าที่ของ Backend

  • รับคำขอ (Request): สร้างและกำหนด Endpoint เพื่อรอรับคำขอจาก Frontend
  • ตรวจสอบข้อมูล (Validate): "ไม่เชื่อใจข้อมูลจาก Client" ต้องตรวจสอบความถูกต้องของข้อมูลที่ส่งมาเสมอ
  • ประมวลผล Logic: จัดการข้อมูล (CRUD) ตามคำขอ เช่น บันทึกลงฐานข้อมูล
  • ส่งคำตอบ (Response): ส่งข้อมูลกลับในรูปแบบ JSON พร้อม Status Code ที่ถูกต้องและสม่ำเสมอ
  • จัดการ CORS: กำหนดนโยบายเพื่ออนุญาตให้ Frontend ที่อยู่คนละ Origin สามารถเรียกใช้ API ได้

กรณีศึกษา: เว็บรีวิวร้านอาหาร

เพื่อให้เห็นภาพชัดเจน ลองดูตัวอย่าง Flow การทำงานเมื่อผู้ใช้ต้องการดูข้อมูลร้านอาหารในโปรเจคที่เราจะทำกัน

1. User Action

ผู้ใช้คลิกที่ "ร้านอาหาร A" ใน React App, UI แสดงสถานะ "กำลังโหลด..."

2. Request

React ส่ง GET request ไปยัง /api/restaurants/1

3. Backend Process

Express Server รับ request, ไปดึงข้อมูลร้านอาหาร id=1 จากไฟล์หรือฐานข้อมูล

4. Response

Server ส่งข้อมูลร้านอาหารกลับไปเป็น JSON: { "name": "ร้านอาหาร A", ... }

5. UI Update

React รับข้อมูล, อัปเดต State, และแสดงผลรายละเอียดของร้านอาหารบนหน้าจอ

Node.js คืออะไร? 🤔

คำจำกัดความง่ายๆ: Node.js คือ สภาพแวดล้อมที่ทำให้เราสามารถรันโค้ด JavaScript นอกเบราว์เซอร์ได้

🌐 ในเบราว์เซอร์ (Chrome, Firefox)

  • HTML, CSS สำหรับแสดงผล
  • JavaScript สำหรับ interaction
  • DOM API สำหรับจัดการหน้าเว็บ

💻 ใน Node.js (คอมพิวเตอร์เรา)

  • JavaScript สำหรับทุกอย่าง
  • File System API สำหรับจัดการไฟล์
  • HTTP API สำหรับสร้างเซิร์ฟเวอร์

ใจความสำคัญ

ใช้ ภาษาเดียวกัน (JavaScript) แต่ทำงานใน สภาพแวดล้อมที่ต่างกัน (Different Environment) และมี พลังมากกว่า (More Powers) เช่น เข้าถึงไฟล์, ฐานข้อมูล, และสร้างเซิร์ฟเวอร์ได้

Node.js vs Browser JavaScript 🥊

ถึงแม้จะใช้ภาษา JavaScript เหมือนกัน แต่ก็มีความแตกต่างที่สำคัญอยู่ครับ

Feature 🌐 Browser 💻 Node.js
Global Object window global
DOM Manipulation ✅ document.getElementById() ❌ ไม่มี (เพราะไม่มีหน้าเว็บ)
File Access ❌ ไม่ได้ (เพื่อความปลอดภัย) ✅ fs module
HTTP Server ❌ ไม่ได้ ✅ http module

🌐 โค้ดใน Browser

// แสดง "Hello" บนหน้าเว็บ
document.getElementById('title').innerHTML = 'Hello World';

💻 โค้ดใน Node.js

// เขียนไฟล์ "hello.txt" ลงในเครื่อง
const fs = require('fs');
fs.writeFileSync('hello.txt', 'Hello World');

NPM คืออะไร? 📦

NPM ย่อมาจาก Node Package Manager

เราอาจเรียกมันง่ายๆ ว่า "App Store สำหรับนักพัฒนา"

NPM คือคลังเก็บโค้ด (เรียกว่า package) ขนาดใหญ่ที่สุดในโลก ที่นักพัฒนาสามารถแบ่งปันและนำโค้ดของคนอื่นมาใช้ในโปรเจคของเราได้

🍎 App Store

  • หาแอปที่ต้องการ
  • ติดตั้ง
  • ใช้งาน

📦 NPM

  • หา package ที่ต้องการ
  • ติดตั้ง (npm install)
  • นำเข้า (import/require)

ปัจจุบันมีมากกว่า 2 ล้าน packages บน NPM ทำให้เราพัฒนาเว็บแอปพลิเคชันได้รวดเร็วอย่างไม่น่าเชื่อ

NPM Package ยอดนิยม 🌟

นี่คือตัวอย่าง packages ที่ถูกดาวน์โหลดหลายสิบล้านครั้งต่อสัปดาห์

  • express → Web framework อันดับ 1 สำหรับสร้างเซิร์ฟเวอร์ (ที่เราจะเรียนต่อ)
  • react → UI library ที่เรากำลังใช้อยู่นั่นเอง!
  • axios → สำหรับการเรียก API จากฝั่ง Frontend ไปยัง Backend
  • lodash → ชุดเครื่องมือช่วยให้เขียน JavaScript ได้สั้นและง่ายขึ้น
  • nodemon → เครื่องมือช่วยรีสตาร์ทเซิร์ฟเวอร์อัตโนมัติเมื่อแก้โค้ด

การติดตั้งก็ง่ายแสนง่าย:

# ติดตั้ง package เดียว
npm install express

# ติดตั้งหลาย packages พร้อมกัน
npm install express body-parser cors

# ติดตั้งสำหรับตอนพัฒนาเท่านั้น
npm install --save-dev nodemon

การติดตั้ง Node.js & NPM 💻

NPM จะถูกติดตั้งมาพร้อมกับ Node.js โดยอัตโนมัติ

  1. ดาวน์โหลด: ไปที่ nodejs.org แล้วเลือกเวอร์ชัน LTS (Long-Term Support) ซึ่งเป็นเวอร์ชันที่เสถียรและแนะนำสำหรับคนส่วนใหญ่
  2. ติดตั้ง: ดับเบิลคลิกไฟล์ที่ดาวน์โหลดมาแล้วกด Next ไปเรื่อยๆ จนเสร็จ
  3. ตรวจสอบการติดตั้ง: เปิดโปรแกรม Terminal (หรือ Command Prompt/PowerShell ใน Windows) แล้วพิมพ์คำสั่ง:
$ node -v
v20.10.0 (หรือเวอร์ชันอื่น)

$ npm -v
10.2.3 (หรือเวอร์ชันอื่น)

ถ้าเห็นเวอร์ชันปรากฏขึ้นมาแบบนี้ แสดงว่าการติดตั้งสำเร็จเรียบร้อยแล้ว! 🎉

16/25

เข้าใจ `package.json` 📄

ไฟล์นี้เปรียบเสมือน "ใบแสดงตัวตน" ของโปรเจค ที่บอกทุกอย่างเกี่ยวกับโปรเจคของเรา

{
  "name": "food-delivery-api",
  "version": "1.0.0",
  "description": "API สำหรับแอปสั่งอาหาร",
  "main": "server.js",
  "scripts": {
    "start": "node server.js",
    "dev": "nodemon server.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.1"
  }
}

ความแตกต่างสำคัญ

dependencies: คือ packages ที่จำเป็นสำหรับให้แอปพลิเคชันทำงานได้บน Production (ตอนใช้งานจริง) เช่น `express`, `react`

devDependencies: คือ packages ที่ใช้สำหรับช่วยในตอนพัฒนาเท่านั้น ไม่ได้ถูกนำไปใช้บน Production เช่น `nodemon` (สำหรับรีสตาร์ทเซิร์ฟเวอร์), `eslint` (สำหรับตรวจจับ error)

Semantic Versioning & node_modules

`^4.18.2`: ตัวเลขเวอร์ชัน (Major.Minor.Patch) สัญลักษณ์ `^` (Caret) หมายความว่า NPM สามารถอัปเดตเวอร์ชัน Minor และ Patch ได้ (เช่น 4.19.0) แต่ห้ามอัปเดตเวอร์ชัน Major (เช่น 5.0.0) เพราะอาจทำให้โค้ดพังได้

NPM Scripts ⚡

scripts ใน `package.json` คือทางลัดสำหรับรันคำสั่งที่ยาวและซับซ้อน ให้เหลือเพียงคำสั่งสั้นๆ ที่จำง่าย

"scripts": {
    "start": "node server.js",
    "dev": "nodemon server.js",
    "test": "jest",
    "lint": "eslint ."
}

การใช้งาน

# แทนที่จะพิมพ์ยาวๆ
$ nodemon server.js
# เราใช้ script แทน
$ npm run dev

ทำไมต้องใช้ Scripts?

  • สะดวก: จำคำสั่งง่ายๆ แทนคำสั่งยาว
  • เป็นมาตรฐาน: ทุกคนในทีมใช้คำสั่งเดียวกัน
  • อัตโนมัติ: สามารถรันหลายคำสั่งต่อกันได้

Scripts คือเวทมนตร์ที่ทำให้ชีวิต developer ง่ายขึ้นอย่างมหาศาล! ✨

Module System ของ Node.js 🧩

แนวคิดของ Module คือ "การแบ่งโค้ดเป็นชิ้นส่วนเล็กๆ เพื่อนำกลับมาใช้ซ้ำและจัดการได้ง่าย"

Node.js มีระบบ Module หลักๆ 2 แบบ:

1. CommonJS (วิธีดั้งเดิม)

// math.js
function add(a, b) { return a + b; }
module.exports = { add };

// app.js
const math = require('./math');
console.log(math.add(5, 3)); // 8

2. ES6 Modules (วิธีใหม่)

// math.mjs
export function add(a, b) { return a + b; }

// app.mjs
import { add } from './math.mjs';
console.log(add(5, 3)); // 8

Built-in Modules ที่สำคัญ 🛠️

Node.js มาพร้อมกับ Module ที่มีประโยชน์มากมายติดตั้งมาให้แล้ว ไม่ต้องลงเพิ่ม

  • File System (`fs`): สำหรับอ่าน, เขียน, ลบไฟล์ 
    fs.writeFileSync('note.txt', 'Hello');
  • Path (`path`): สำหรับจัดการ path ของไฟล์และโฟลเดอร์ 
    path.join(__dirname, 'data.json');
  • OS (`os`): สำหรับดูข้อมูลระบบปฏิบัติการ 
    console.log(os.cpus().length);
  • URL (`url`): สำหรับจัดการและแยกส่วน URL 
    new URL(myUrl).hostname;

และที่สำคัญที่สุดคือ HTTP (`http`) สำหรับสร้างเว็บเซิร์ฟเวอร์ ซึ่งเป็นหัวใจของ Backend! 💪

HTTP Module - เซิร์ฟเวอร์แรก! 🌐 (Interactive)

เรามาสร้างเว็บเซิร์ฟเวอร์จริงๆ ด้วยโค้ดไม่กี่บรรทัดกันครับ

📝 โค้ด `server.js`
const http = require('http');

const server = http.createServer((req, res) => {
    if (req.url === '/') {
        res.end('<h1>หน้าแรก</h1><a href="/about">ไปหน้าเกี่ยวกับเรา</a>');
    } else if (req.url === '/about') {
        res.end('<h1>เกี่ยวกับเรา</h1><a href="/">กลับหน้าแรก</a>');
    } else {
        res.writeHead(404);
        res.end('<h1>404 Not Found</h1>');
    }
});

server.listen(3000, () => {
    console.log('🚀 Server is running on http://localhost:3000');
});
👀 ผลลัพธ์ (จำลอง)
http://localhost:3000/

Nodemon - Development Tool 🔄

เคยเจอปัญหานี้ไหมครับ: แก้โค้ด → บันทึก → ต้องหยุดเซิร์ฟเวอร์ (Ctrl+C) → แล้วรันใหม่ วนไปเรื่อยๆ...

Nodemon คือทางออก!

Nodemon คือเครื่องมือที่จะคอย "จับตาดู" ไฟล์โปรเจคของเรา และเมื่อไหร่ก็ตามที่มีการเปลี่ยนแปลงและบันทึกไฟล์ มันจะทำการรีสตาร์ทเซิร์ฟเวอร์ให้เราโดยอัตโนมัติ!

1. ติดตั้ง

npm install --save-dev nodemon

2. เพิ่ม Script ใน `package.json`

"scripts": {
  "start": "node server.js",
  "dev": "nodemon server.js"
}

3. รันด้วยคำสั่งใหม่

npm run dev

เพียงเท่านี้ ชีวิตการพัฒนา Backend ของเราก็จะง่ายขึ้นมหาศาล ไม่ต้องเสียสมาธิกับการรีสตาร์ทเซิร์ฟเวอร์เองอีกต่อไป! ✅

สรุปและเตรียมตัวสำหรับ Express.js 🎯

สิ่งที่เราเรียนรู้ในหัวข้อนี้:

  • Node.js: คือ JavaScript runtime ที่ทำงานนอกเบราว์เซอร์
  • NPM: คือ "App Store" ของนักพัฒนา สำหรับจัดการ packages
  • Modules: วิธีการแบ่งโค้ดเป็นส่วนๆ เพื่อจัดการง่าย (require, import)
  • Built-in APIs: พลังในการเข้าถึงไฟล์ (fs) และสร้างเซิร์ฟเวอร์ (http)

แต่ยังมีปัญหาอยู่... 🤔

การสร้างเซิร์ฟเวอร์ด้วย `http` module พื้นฐานนั้นค่อนข้างซับซ้อนเมื่อมีหลาย URL (ดูโค้ดในสไลด์ 21)

// ต้องใช้ if/else ซ้อนกันเยอะมาก...
if (req.url === '/api/users' && req.method === 'GET') {
    // ...
} else if (req.url === '/api/users' && req.method === 'POST') {
    // ...
} // และอื่นๆ อีกมากมาย... 😵

Express.js จะเข้ามาเป็นฮีโร่ 🦸‍♂️ ที่ช่วยให้การเขียนโค้ดส่วนนี้ง่ายขึ้น สั้นลง และเป็นระเบียบอย่างมาก ซึ่งเราจะเรียนกันในหัวข้อถัดไป!
