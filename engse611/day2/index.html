สรุปการเรียนรู้วันที่ 2: Making Websites Interactive

ภาพรวมการเดินทางในวันที่ 2 ของเรา: จากเว็บนิ่งๆ สู่เว็บแอปพลิเคชันที่ "มีชีวิต" สามารถรับฟังและโต้ตอบกับการกระทำของผู้ใช้ได้

1. JavaScript & DOM API: สมองและแขนขาของเว็บ

เราเริ่มต้นด้วยการเรียนรู้ภาษา JavaScript ซึ่งเป็น "สมอง" ที่ใช้เขียน Logic และคำสั่งต่างๆ จากนั้นเรียนรู้วิธีใช้ "แขนขา" อย่าง DOM API เพื่อเข้าไปควบคุมและเปลี่ยนแปลงส่วนต่างๆ ของหน้าเว็บ (HTML) ได้

2. Event Handling: การ "ดักฟัง" และ "โต้ตอบ"

เมื่อเว็บของเรามีสมองและแขนขาแล้ว ขั้นต่อไปคือการเพิ่ม "หู" เพื่อดักฟังการกระทำของผู้ใช้ (Events) เช่น การคลิก, การพิมพ์, การเลื่อนเมาส์ แล้วเขียนฟังก์ชันเพื่อ "โต้ตอบ" กับการกระทำเหล่านั้น

(เช่น Click)

("ดักฟัง" ที่ปุ่ม)

("โต้ตอบ")

3. Form Validation & Storage: การสร้างความน่าเชื่อถือและความทรงจำ

สุดท้าย เราทำให้เว็บฉลาดขึ้นโดยการตรวจสอบข้อมูลที่ผู้ใช้กรอก (Validation) เพื่อป้องกันข้อมูลผิดพลาด และใช้ Local Storage เพื่อให้เว็บสามารถ "จดจำ" ข้อมูลสำคัญได้ แม้จะปิดเบราว์เซอร์ไปแล้ว
